4. **Create `scripts/screenshots.ts`** -- a Playwright automation script that:
   - Imports `chromium` from `playwright` (NOT `@playwright/test` -- this is a standalone script, not a test)
   - Defines screens to capture: create page (`/`) and reveal page (requires creating a test secret first via the API)
   - Uses fixed viewport 1280x800 for consistency
   - Uses `colorScheme: 'dark'` for visual impact per user decision
   - For the reveal screenshot: uses the API to create a secret (POST /api/secrets with pre-encrypted ciphertext), then navigates to the reveal URL with an interstitial click
   - Saves PNGs to `docs/screenshots/create-dark.png` and `docs/screenshots/reveal-dark.png`
   - The base URL should default to `http://localhost:3000` (configurable via env var)
   - Wait for `networkidle` load state before each screenshot
   - For the create page: wait a beat after load for any animations to settle, then screenshot
   - For the reveal page: the flow is (a) create a secret via direct API call with dummy encrypted data, (b) navigate to `/secret/{id}#{key}`, (c) click the "View Secret" interstitial button, (d) wait for the decrypted content to appear, (e) screenshot. NOTE: The crypto module must encrypt real data since the reveal page decrypts in-browser. Import `encrypt` from the client crypto module (`../client/src/crypto/index.ts`). Encrypt a sample message like "This is a secret message shared securely with SecureShare." and use the resulting ciphertext and key.
   - Run with: `npx tsx scripts/screenshots.ts`

5. **Run the screenshot script:** The dev servers must be running for this. Start the dev server stack (`npm run dev:server` in background, ensure PostgreSQL is available), then run `npx tsx scripts/screenshots.ts`. If the dev servers are not available, use Playwright's built-in webServer capability or start the servers programmatically within the script (similar to how the E2E config does it -- check `e2e/playwright.config.ts` for the webServer pattern). The simplest approach: start servers with child_process, wait for ready, capture, kill.

   IMPORTANT: Verify the screenshots exist in `docs/screenshots/` after running. Verify `git status` shows them as trackable (not ignored).
