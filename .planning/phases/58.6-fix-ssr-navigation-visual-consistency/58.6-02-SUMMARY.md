## Decisions Made
- SVG strings embedded verbatim as JS single-quoted string literals inside the script tag — TypeScript module constants (MOON_SVG, SUN_SVG, MONITOR_SVG) are server-time only and cannot be referenced at runtime
- updateSummaryIcon() always re-reads localStorage rather than closing over the stored variable — correct in both load-time call and click-handler call paths
- Test assertions search script block content for unique SVG attribute substrings (`circle cx=` and `rect width=`) to avoid false positives from SVG already present in the nav button markup

## Deviations from Plan

**1. [Rule 1 - Bug] Test assertion strings corrected after prettier formatting changed escape sequences**
- **Found during:** RED phase test authoring
- **Issue:** Initial test used `'width=\\"20\\"...'` (escaped double-quotes), but the SVG content embedded in the JS script has literal double-quotes (no escaping needed in single-quoted JS strings). Prettier reformatted the assertions, preserving the escape sequences, making them search for backslash+quote sequences that would never appear.
- **Fix:** Changed to unique substring patterns (`'rect width='` and `'circle cx='`) that are unambiguous within the script block
- **Files modified:** server/src/routes/seo/templates/layout.test.ts
