1. **`server/src/middleware/validate.ts`** -- Zod validation middleware factory:
   - `validateBody<T>(schema: ZodSchema<T>)`: returns Express middleware that calls `schema.safeParse(req.body)`. On failure, respond 400 with `{ error: 'validation_error', details: result.error.flatten() }`. On success, assign `req.body = result.data` and call `next()`.
   - `validateParams<T>(schema: ZodSchema<T>)`: same pattern but for `req.params`. On failure, respond 400. On success, call `next()`.
   - Import types from express (`Request, Response, NextFunction`).
   - Use `ZodSchema` from zod (not `z.ZodType` -- zod 4.x changed this; check the import).

2. **`server/src/middleware/error-handler.ts`** -- Global error handler:
   - Import `logger` from `./logger.js`
   - Standard Express error middleware signature (4 params: err, req, res, next)
   - Log the error using `logger.error({ err: { message: err.message, stack: err.stack } }, 'Unhandled error')` -- do NOT log request details (could contain secret IDs)
   - Respond 500 with `{ error: 'internal_error', message: 'An unexpected error occurred.' }`
   - Never leak stack traces or internal error details to the client

3. **`server/src/services/secrets.service.ts`** -- Core business logic:

   **`createSecret(ciphertext: string, expiresIn: '1h' | '24h' | '7d' | '30d')`:**
   - Calculate `expiresAt` from current time + duration. Map: 1h=3600000ms, 24h=86400000ms, 7d=604800000ms, 30d=2592000000ms
   - Insert into secrets table using `db.insert(secrets).values({ ciphertext, expiresAt }).returning()`
   - Return the inserted row (contains nanoid ID, expiresAt, createdAt)

   **`retrieveAndDestroy(id: string)`:**
   - Use `db.transaction(async (tx) => { ... })` for atomicity
   - Step 1 (SELECT): `tx.select().from(secrets).where(eq(secrets.id, id))` -- get the secret
   - If not found, return null (already consumed, never existed, or expired)
   - Step 2 (ZERO): `tx.update(secrets).set({ ciphertext: '\x00'.repeat(secret.ciphertext.length) }).where(eq(secrets.id, id))` -- overwrite ciphertext with null bytes before deletion (SECR-08 data remanence mitigation)
   - Step 3 (DELETE): `tx.delete(secrets).where(eq(secrets.id, id))` -- remove the row
   - Return the original secret object (from Step 1, which has the real ciphertext)
   - Import `eq` from `drizzle-orm`, `db` from `../db/connection.js`, `secrets` from `../db/schema.js`
  </action>

  </verify>

   ```typescript
   const SECRET_NOT_AVAILABLE = {
     error: 'not_found',
     message: 'This secret does not exist, has already been viewed, or has expired.',
   } as const;
   ```

   **POST `/` (mounted at /api/secrets):**
   - Middleware chain: `validateBody(CreateSecretSchema)`
   - Handler: call `createSecret(req.body.ciphertext, req.body.expiresIn)`
   - Respond 201 with `{ id: secret.id, expiresAt: secret.expiresAt.toISOString() }`

   **GET `/:id`:**
   - Middleware chain: `validateParams(SecretIdParamSchema)`
   - Handler: call `retrieveAndDestroy(req.params.id)`
   - If null: respond 404 with `SECRET_NOT_AVAILABLE` (same response for nonexistent, expired, already-viewed)
   - If found: respond 200 with `{ ciphertext: secret.ciphertext, expiresAt: secret.expiresAt.toISOString() }`
   - Use Express 5 native async error handling (async handlers, no try-catch wrappers needed)

   Export the router as `secretsRouter`.
