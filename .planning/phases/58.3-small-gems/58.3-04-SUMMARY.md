# Phase 58.3 Plan 04: Burn Timer Countdown Summary

**setInterval countdown in reveal.ts that ticks "Content hides in Ns" below the terminal block and replaces pre content at zero when ?burn= param is present in the reveal URL**

## Accomplishments

- Added `attachBurnTimer(container, seconds)` as a module-level exported function in reveal.ts
- `renderRevealedSecret` reads `?burn=` from `window.location.search` after appending wrapper to container and calls `attachBurnTimer` automatically
- Status line inserted after `.mb-6` terminal element with text "Content hides in Ns" (suppressed under prefers-reduced-motion)
- At zero: `<pre>` textContent replaced with `[Content hidden — copy it before closing]`, status line shows "Content has been hidden."
- `terminalEl.isConnected` guard clears interval silently on SPA navigation
- All 6 reveal.test.ts tests pass (4 existing + 2 burn timer Wave 0 tests)

## Task Commits

1. **Task 1: Add burn timer countdown to reveal.ts** - `a90f26c` (feat)

## Files Created/Modified

- `client/src/pages/reveal.ts` - Added `attachBurnTimer()` function and burn timer call in `renderRevealedSecret`

## Decisions Made

- `attachBurnTimer` reads `window.location.search` inside `renderRevealedSecret` rather than only being called from `renderRevealPage` — this is required for testability since the Wave 0 tests call `renderRevealedSecret` directly with a stubbed `window.location`
- Used `querySelector('.mb-6')` to find the terminal block (works in test env where `createTerminalBlock` is mocked to return a plain `<div>` that gets `mb-6` added by `renderRevealedSecret`)
- Used `querySelector('pre')` only for the content-hiding step at T=0, with a null guard — graceful no-op in test environments
- `burnSeconds > 0` guard: `parseInt('abc', 10)` returns NaN; `!Number.isNaN` filters it; negative/zero values also rejected
