     reportsDirectory: './coverage',

<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow with lint, test, and E2E jobs</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` with the following structure. This is a single workflow with 3 jobs forming a dependency chain where lint gates everything.

**Workflow triggers:**
```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
```

**Job 1: lint** (runs first, CICD-04 fail-fast)
- `runs-on: ubuntu-latest`
- Steps: checkout@v6, setup-node@v6 (node 24, cache: npm), `npm ci`, `npm run lint`, `npm run format:check`

**Job 2: test** (needs: [lint], CICD-01)
- `runs-on: ubuntu-latest`
- `needs: [lint]` — only runs if lint passes (fail-fast, CICD-04)
- Services: PostgreSQL 17 with health check (server integration tests require real DB)
  ```yaml
  services:
    postgres:
      image: postgres:17
      env:
        POSTGRES_USER: secureshare
        POSTGRES_PASSWORD: secureshare
        POSTGRES_DB: secureshare
      options: >-
        --health-cmd pg_isready
        --health-interval 10s
        --health-timeout 5s
        --health-retries 5
      ports:
        - 5432:5432
  ```
- Job-level env:
  ```yaml
  env:
    DATABASE_URL: postgresql://secureshare:secureshare@localhost:5432/secureshare
    NODE_ENV: test
  ```
- Steps: checkout@v6, setup-node@v6 (node 24, cache: npm), `npm ci`, run migrations (`node --import tsx server/src/db/migrate.ts`), `npx vitest run --coverage`
- The explicit migration step ensures the DB schema exists before Vitest server tests run. Without it, server integration tests would fail with "relation does not exist" errors.

**Job 3: e2e** (needs: [lint], CICD-02)
- `runs-on: ubuntu-latest`
- `needs: [lint]` — runs in parallel with test job after lint passes
- Services: PostgreSQL 17 AND Redis 7, both with health checks
  ```yaml
  services:
    postgres:
      image: postgres:17
      env:
        POSTGRES_USER: secureshare
        POSTGRES_PASSWORD: secureshare
        POSTGRES_DB: secureshare
      options: >-
        --health-cmd pg_isready
        --health-interval 10s
        --health-timeout 5s
        --health-retries 5
      ports:
        - 5432:5432
    redis:
      image: redis:7
      options: >-
        --health-cmd "redis-cli ping"
        --health-interval 10s
        --health-timeout 5s
        --health-retries 5
      ports:
        - 6379:6379
  ```
- Job-level env:
  ```yaml
  env:
    DATABASE_URL: postgresql://secureshare:secureshare@localhost:5432/secureshare
    REDIS_URL: redis://localhost:6379
  ```
- Steps:
  1. checkout@v6
  2. setup-node@v6 (node 24, cache: npm)
  3. `npm ci`
  4. Cache Playwright browsers using actions/cache@v4:
     ```yaml
     - name: Cache Playwright browsers
       id: playwright-cache
       uses: actions/cache@v4
       with:
         path: ~/.cache/ms-playwright
         key: playwright-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
     ```
  5. Install Playwright Chromium (conditional on cache):
     ```yaml
     - name: Install Playwright browsers
       if: steps.playwright-cache.outputs.cache-hit != 'true'
       run: npx playwright install --with-deps chromium
     - name: Install Playwright system deps
       if: steps.playwright-cache.outputs.cache-hit == 'true'
       run: npx playwright install-deps chromium
     ```
     When cache hits, browsers exist but system deps may not (fresh runner). `install-deps` installs only OS libraries.
  6. Run E2E tests — Chromium only per locked decision:
     ```yaml
     - name: Run E2E tests
       run: npx playwright test --config e2e/playwright.config.ts --project=chromium
     ```
     The Playwright config already sets `E2E_TEST=true` in webServer.env, `CI=true` is set by GitHub Actions automatically, and webServer handles building and starting the server + running migrations.
  7. Upload Playwright report on failure:
     ```yaml
     - name: Upload Playwright report
       if: ${{ !cancelled() }}
       uses: actions/upload-artifact@v6
       with:
         name: playwright-report
         path: playwright-report/
         retention-days: 7
     ```

**Key design decisions (Claude's discretion):**
- Single workflow file (simple, one badge)
- Node 24 only (project targets 24.x LTS specifically)
- test and e2e run in parallel after lint (saves time)
- No explicit deploy job (Render handles via autoDeployTrigger, covered in Plan 02)
- No Playwright sharding (4 test files, not worth the overhead)
- GitHub reporter for E2E (already configured in playwright.config.ts via `process.env.CI`)
- Upload Playwright report on any non-cancelled run for debugging
  </action>
