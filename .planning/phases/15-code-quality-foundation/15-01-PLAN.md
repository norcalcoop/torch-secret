<task type="auto">
  <name>Task 1: Install dev dependencies and create ESLint 10 flat config</name>
  <files>
    package.json
    eslint.config.ts
  </files>
  <action>
1. Install all code quality dev dependencies in one command:
   ```bash
   npm install --save-dev eslint @eslint/js typescript-eslint eslint-config-prettier prettier prettier-plugin-tailwindcss husky lint-staged globals
   ```

2. Create `eslint.config.ts` at project root with:
   - Import `defineConfig` and `globalIgnores` from `'eslint/config'`
   - Import `eslint` from `'@eslint/js'`
   - Import `tseslint` from `'typescript-eslint'`
   - Import `eslintConfigPrettier` from `'eslint-config-prettier/flat'`
   - Import `globals` from `'globals'`
   - `globalIgnores(['dist', 'client/dist', 'drizzle', 'node_modules'])` first
   - `eslint.configs.recommended` for base rules
   - `tseslint.configs.recommendedTypeChecked` for TypeScript type-aware rules
   - Language options block with `parserOptions: { projectService: true, tsconfigRootDir: import.meta.dirname }`
   - Client-specific block: `files: ['client/src/**/*.ts']` with `languageOptions: { globals: globals.browser }`
   - Server-specific block: `files: ['server/src/**/*.ts']` with `languageOptions: { globals: globals.node }`
   - Shared block: `files: ['shared/**/*.ts']` with `languageOptions: { globals: globals.node }` (shared types used server-side)
   - Config files block: `files: ['*.config.ts', '*.config.mjs']` with `extends: [tseslint.configs.disableTypeChecked]` (these files aren't in any tsconfig include)
   - JS files block: `files: ['**/*.js', '**/*.mjs', '**/*.cjs']` with `extends: [tseslint.configs.disableTypeChecked]`
   - `eslintConfigPrettier` LAST to disable conflicting rules

3. Add npm scripts to package.json (do NOT remove existing scripts):
   - `"lint": "eslint ."`
   - `"lint:fix": "eslint . --fix"`
   - `"format": "prettier --write ."`
   - `"format:check": "prettier --check ."`
   - `"prepare": "husky"`

Note: `eslint.config.ts` is supported natively in ESLint 10 (jiti is auto-resolved). Do NOT use `tseslint.config()` — it is deprecated. Use `defineConfig()` from `'eslint/config'` instead.
  </action>
  <verify>
Run `npx eslint --print-config client/src/app.ts 2>&1 | head -5` to confirm ESLint loads the config without errors. The output should show JSON config (not an error about missing config).
  </verify>
  <done>eslint.config.ts exists with defineConfig, type-aware rules, environment-specific globals, and Prettier compatibility. package.json has lint, lint:fix, format, format:check, and prepare scripts.</done>
</task>

<task type="auto">
  <name>Task 2: Create Prettier config, ignore file, lint-staged config, and Husky pre-commit hook</name>
  <files>
    .prettierrc.json
    .prettierignore
    .lintstagedrc.json
    .husky/pre-commit
  </files>
  <action>
1. Create `.prettierrc.json` at project root:
   ```json
   {
     "singleQuote": true,
     "trailingComma": "all",
     "printWidth": 100,
     "tabWidth": 2,
     "semi": true,
     "plugins": ["prettier-plugin-tailwindcss"],
     "tailwindStylesheet": "./client/src/styles.css"
   }
   ```

2. Create `.prettierignore` at project root:
   ```
   dist
   client/dist
   node_modules
   package-lock.json
   drizzle
   *.png
   *.ico
   *.svg
   .planning
   ```
   Rationale: Skip generated files (dist, drizzle migrations), binary assets, lock file, and planning docs (markdown formatting would conflict with GSD workflow templates).

3. Create `.lintstagedrc.json` at project root:
   ```json
   {
     "*.{ts,js,mjs,cjs}": ["eslint --fix", "prettier --write"],
     "*.{json,css,md,html}": ["prettier --write"]
   }
   ```
   ESLint runs first (may auto-fix lint issues), then Prettier reformats. Order matters.

4. Initialize Husky:
   ```bash
   npx husky init
   ```
   This creates `.husky/` directory and adds `"prepare": "husky"` to package.json (already added in Task 1, so verify it's there).

5. Overwrite `.husky/pre-commit` content (husky init creates a default with `npm test`):
   ```shell
   npx lint-staged
   ```
   Just that one line — no shebang needed in Husky v9.
  </action>
  <verify>
1. Verify `.prettierrc.json` is valid JSON: `node -e "require('./.prettierrc.json')"` (no error = valid)
2. Verify `.lintstagedrc.json` is valid JSON: `node -e "require('./.lintstagedrc.json')"` (no error = valid)
3. Verify `.husky/pre-commit` exists and contains `lint-staged`: `cat .husky/pre-commit`
4. Verify Prettier config is loaded: `npx prettier --find-config-path client/src/app.ts` (should output `.prettierrc.json`)
  </verify>

</task>
