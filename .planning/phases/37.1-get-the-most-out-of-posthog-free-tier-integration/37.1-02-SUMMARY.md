## Decisions Made

- `getActiveTabId()` returns raw tab enum — separate from `getProtectionType()` which collapses generate+custom to 'password' for the API. Analytics needs the 4-way distinction; the API uses the 3-way protection type. Two accessors, two purposes.
- `analyticsProtectionType` computed in the submit handler with an explicit ternary chain rather than inside `getActiveTabId()` — keeps the analytics mapping concern at the call site, not inside the UI component.
- `registeredAt` declared as `let` with `undefined` default outside the try block — on getMe() failure, `identifyUser` receives `undefined` for registeredAt (safe degradation, PostHog accepts undefined optional props).
- `captureCheckoutInitiated` fires synchronously before `window.location.href = url` — PostHog batches/flushes before unload events, so the event reliably reaches the server even with immediate page navigation.
