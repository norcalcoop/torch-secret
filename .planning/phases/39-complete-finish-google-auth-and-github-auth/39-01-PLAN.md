
**Better Auth callback URL pattern:**
Better Auth auto-generates the OAuth callback at `{BETTER_AUTH_URL}/api/auth/callback/google`.
- Dev: `BETTER_AUTH_URL=http://localhost:3000` → callback is `http://localhost:3000/api/auth/callback/google`
- Prod: `BETTER_AUTH_URL=https://torchsecret.com` → callback is `https://torchsecret.com/api/auth/callback/google`

**Credential storage convention (Phase 37.2):** All secrets go into Infisical only. Do NOT add to .env files.

**No-trailing-slash rule:** Register redirect URIs exactly as shown — no trailing slash. A trailing slash is the #1 cause of `redirect_uri_mismatch` errors.

  </what-needs-doing>
  <steps>
    1. Go to https://console.cloud.google.com/apis/credentials
    2. Select or create a project (e.g., "Torch Secret")
    3. Click **Create Credentials** → **OAuth client ID**
    4. Choose **Web application**
    5. Set Name: `Torch Secret`
    6. Under **Authorized JavaScript origins**, add BOTH:
       - `http://localhost:3000`
       - `https://torchsecret.com`
    7. Under **Authorized redirect URIs**, add BOTH:
       - `http://localhost:3000/api/auth/callback/google`
       - `https://torchsecret.com/api/auth/callback/google`
    8. Click **Create**
    9. Copy the **Client ID** and **Client Secret** — you'll need them in Task 2

    IMPORTANT: No trailing slashes on any URI. No port number on the production URIs.
  </steps>

  </resume-signal>
