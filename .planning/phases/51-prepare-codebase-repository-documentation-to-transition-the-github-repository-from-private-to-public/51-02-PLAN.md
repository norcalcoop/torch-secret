  <action>
Add `if: ${{ !github.event.pull_request.head.repo.fork }}` to the `test` job and the `e2e` job, inserted immediately after the `needs: [lint]` line in each.

The result for both jobs:

```yaml
test:
  name: Test
  runs-on: ubuntu-latest
  needs: [lint]
  if: ${{ !github.event.pull_request.head.repo.fork }}
```

```yaml
e2e:
  name: E2E
  runs-on: ubuntu-latest
  needs: [lint]
  if: ${{ !github.event.pull_request.head.repo.fork }}
```

**Why this condition works correctly across all trigger types:**
- Own-repo PR: `head.repo.fork` is false → `!false` = true → jobs run
- Fork PR: `head.repo.fork` is true → `!true` = false → jobs are skipped (shown as grey/skipped, not red/failed)
- Push to main: `github.event.pull_request` is null → null path evaluates to false in GitHub Actions expression syntax → `!false` = true → jobs run
- `workflow_dispatch`: same as push — `github.event.pull_request` is null → jobs run

Do NOT use `== false` comparison syntax (e.g., `github.event.pull_request.head.repo.fork == false`) — the negation form `!github.event.pull_request.head.repo.fork` is the correct pattern per research.

Do NOT add an if: condition to the `docker-build` job — it has no Infisical dependency and should run on all PRs including forks.

Do NOT add an if: condition to the `lint` job — it must always run.

Only two lines are added to the file. No other changes to ci.yml.
