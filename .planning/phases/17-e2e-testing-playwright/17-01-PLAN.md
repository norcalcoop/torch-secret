**Step 5: Create `e2e/fixtures/crypto-helpers.ts`.**
This module MUST replicate the exact encryption pipeline from the browser:
1. UTF-8 encode the plaintext via `TextEncoder`
2. Apply PADME padding: replicate `padmeLength()` and `padPlaintext()` from `client/src/crypto/padding.ts` exactly. `MIN_PADDED_SIZE = 256`. The format is `[4-byte uint32 BE original length][original data][zero fill to padmeLength]`. The `padmeLength` algorithm: if `len <= 256` return 256; else compute `e = Math.floor(Math.log2(len))`, `s = Math.floor(Math.log2(e)) + 1`, `lastBits = e - s`, `bitMask = (1 << lastBits) - 1`, return `(len + bitMask) & ~bitMask`.
3. Generate a fresh 256-bit key via `crypto.getRandomValues(new Uint8Array(32))`
4. Import as AES-GCM key with `crypto.subtle.importKey('raw', keyData, { name: 'AES-GCM' }, true, ['encrypt'])`
5. Generate a fresh 96-bit IV via `crypto.getRandomValues(new Uint8Array(12))`
6. Encrypt the PADDED bytes (not the raw plaintext bytes) with AES-GCM
7. Combine `[IV 12 bytes][ciphertext + auth tag]`, base64 encode with `Buffer.from(combined).toString('base64')`
8. Export key to base64url: `Buffer.from(rawKey).toString('base64').replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '')`
9. Export a function `encryptForTest(plaintext: string): Promise<{ ciphertext: string; keyBase64Url: string }>`.

CRITICAL: The browser's `decrypt.ts` calls `unpadPlaintext()` after decryption. If the fixture does NOT apply PADME padding before encryption, `unpadPlaintext()` will read garbage from the first 4 bytes (it expects a uint32 BE length prefix) and produce corrupted output. PADME padding is mandatory.

**Test 1: "create a secret via the browser UI"**
1. `await page.goto('/')` -- navigate to home page
2. Wait for the heading "Share a Secret" to be visible (confirms page loaded)
3. Fill the textarea labeled "Your secret" with `'My super secret E2E message'`
4. Click the button "Create Secure Link"
5. Wait for the confirmation heading "Your Secure Link is Ready" to be visible
6. Extract the share URL from the `code` element: `const shareUrl = await page.locator('code').textContent()` -- this returns the full URL like `http://localhost:3000/secret/{id}#{key}`
7. Assert shareUrl contains `/secret/` and contains `#`
8. Store shareUrl in a variable shared across the serial describe block: `let shareUrl: string`

**Test 2: "reveal the secret via the share URL"**
1. `await page.goto(shareUrl)` -- navigate to the share URL (Playwright sends the fragment)
2. Wait for the interstitial heading: `await expect(page.getByRole('heading', { name: "You've received a secret" })).toBeVisible()`
3. Click the "Reveal Secret" button: `await page.getByRole('button', { name: 'Reveal Secret' }).click()`
4. Wait for the reveal heading: `await expect(page.getByRole('heading', { name: 'Secret Revealed' })).toBeVisible()`
5. Assert the decrypted text is visible: `await expect(page.getByText('My super secret E2E message')).toBeVisible()`

**Test 3: "secret is destroyed after viewing"**
1. `await page.goto(shareUrl)` -- navigate to the same URL again
2. The page should show either "Secret Not Available" or "Secret Already Viewed" heading. Wait for one of them: `await expect(page.getByRole('heading', { name: /Secret (Not Available|Already Viewed)/ })).toBeVisible()`

IMPORTANT NOTES:
- The three tests MUST be in a `test.describe.serial` block so they execute in order and share state.
- The `shareUrl` variable is declared in the describe scope and assigned in test 1, read in tests 2 and 3.
- Do NOT use `toHaveURL()` with fragments -- `page.url()` strips the hash.
- The confirmation page is state-based (URL stays `/`), so look for the heading, not the URL.
- The reveal page strips the hash immediately via `history.replaceState`, so do not assert on hash after navigation.
