<task type="auto">
  <name>Task 1: Create forgot-password and reset-password pages</name>
  <files>client/src/pages/forgot-password.ts, client/src/pages/reset-password.ts</files>
  <action>
    MANDATORY: Before writing any code in this task, invoke the `frontend-design` skill.

    1. Create client/src/pages/forgot-password.ts.
       Export renderForgotPasswordPage(container: HTMLElement): Promise<void>.

       Page structure:
       - Header: h1 "Reset Password", subtext "Enter your email to receive a reset link"
       - Card: glassmorphism surface
       - Form: email input (required) + submit button "Send Reset Link"
       - Success state: "Check your email — If an account exists for <email>, you'll receive a
         password reset link shortly." (same message whether account exists or not — prevents
         email enumeration)
       - Error state: show error.message or generic "Failed to send reset email. Please try again."
       - Back link: "Back to sign in" navigates to /login

       Reset flow:
         const { error } = await authClient.requestPasswordReset({
           email,
           redirectTo: '/reset-password',  // Better Auth appends the token as ?token=
         });
         On error: show error message.
         On success (no error): show the success state (do NOT navigate — user needs to check email).
         IMPORTANT: Use authClient.requestPasswordReset() — NOT authClient.forgotPassword()
         (forgotPassword was renamed in Better Auth 1.4 and does not exist).

    2. Create client/src/pages/reset-password.ts.
       Export renderResetPasswordPage(container: HTMLElement): Promise<void>.

       Page structure:
       - Header: h1 "Set New Password", subtext "Enter your new password below"
       - Card: glassmorphism surface
       - On mount: extract token from URL. Better Auth provides the token as a query parameter.
         The URL will be /reset-password?token=<token> because requestPasswordReset was called
         with redirectTo: '/reset-password'.
         const token = new URLSearchParams(window.location.search).get('token');
         If token is missing or null: show an error card "This reset link is invalid or has expired.
         Request a new one." with a link to /forgot-password.
       - Form: new password input (type="password", required, minLength=8) +
               confirm password input (type="password", required) +
               submit button "Reset Password"
       - Client-side validation: confirm passwords match before submitting.
       - Success state: "Password reset successfully. You can now sign in with your new password."
         with a link to /login.
       - Error state: show error.message or "Failed to reset password. The link may have expired."

       Reset submission flow:
         const { error } = await authClient.resetPassword({
           newPassword: password,
           token: token!,
         });
         On error: show error message.
         On success (no error): show success state.

       Accessibility: Both pages use role="alert" on error/success messages.
       Loading state: disable button while awaiting API call.
  </action>
