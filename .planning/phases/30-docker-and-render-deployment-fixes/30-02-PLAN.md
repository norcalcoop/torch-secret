<task type="auto">
  <name>Task 2: Add docker-build CI job to catch Dockerfile regressions</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Add a `docker-build` job to `.github/workflows/ci.yml`. This job runs `docker build` to verify the Dockerfile compiles successfully. It catches regressions (broken COPY paths, npm ci failures, etc.) before they reach Render.

The existing CI structure is:
- `lint` job (no deps)
- `test` job (needs: [lint])
- `e2e` job (needs: [lint])

Add a `docker-build` job that:
- Runs on `ubuntu-latest`
- Has `needs: [lint]` (same as test and e2e — runs after lint passes, not blocking test/e2e)
- Uses `actions/checkout@v4`
- Runs `docker build --no-cache -t secureshare:ci-test .`
- Passes empty strings for VITE_ build args (analytics disabled in CI builds — correct default)

The job definition to add (insert after the `e2e` job, before the end of the file):

```yaml
  docker-build:
    name: Docker Build
    runs-on: ubuntu-latest
    needs: [lint]
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Build Docker image
        run: |
          docker build --no-cache \
            --build-arg VITE_POSTHOG_KEY="" \
            --build-arg VITE_POSTHOG_HOST="" \
            -t secureshare:ci-test \
            .
```

Notes on the implementation:
- `--no-cache` ensures a clean build that reflects the actual Dockerfile state, not a cached layer
- `--build-arg VITE_POSTHOG_KEY=""` satisfies the ARG declarations added in Plan 01 — Vite treats an empty string as undefined and skips PostHog initialization (graceful no-op)
- No push step — this is a validation-only build; no registry authentication needed
- The job does NOT push the image anywhere — it only validates the build succeeds
- `needs: [lint]` means the job runs in parallel with `test` and `e2e` (after lint), not blocking them
- The image tag `secureshare:ci-test` is local only and discarded after the job completes

Insert this job at the end of the `jobs:` block, after the existing `e2e` job definition.
