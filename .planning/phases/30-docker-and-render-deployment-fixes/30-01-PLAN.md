Add the missing auth env vars to the `app` service's `environment` section. The current environment has: DATABASE_URL, REDIS_URL, NODE_ENV, FORCE_HTTPS, PORT, LOG_LEVEL. It is missing BETTER_AUTH_SECRET, BETTER_AUTH_URL, RESEND_API_KEY, and RESEND_FROM_EMAIL — these cause a Zod startup crash.

```yaml
      BETTER_AUTH_URL: 'http://localhost:3000'

      # GITHUB_CLIENT_ID:
      # GITHUB_CLIENT_SECRET:
      # --- PostHog is a build-time var; set VITE_POSTHOG_KEY before docker build ---
      # VITE_POSTHOG_KEY is baked in during docker build — not a runtime env var
```

**Dockerfile changes:**

Add ARG declarations for VITE_ vars in Stage 2 (the build stage), immediately before `RUN npm run build:client`. The current Stage 2 is:

```dockerfile
FROM deps AS build
WORKDIR /app
COPY . .
RUN npm run build:client
```

Change it to:

```dockerfile
FROM deps AS build
WORKDIR /app
COPY . .
# PostHog analytics are baked into the JS bundle at build time.
# Pass via --build-arg or set in Render dashboard (Render auto-passes env vars as build args).
# If not provided, analytics are silently disabled — safe default for local Docker builds.
ARG VITE_POSTHOG_KEY
ARG VITE_POSTHOG_HOST
RUN npm run build:client
```

The ARG declarations with no default value mean:
- If the build arg is not passed, the env var is undefined (empty string in Vite = analytics disabled — correct graceful no-op)
- If passed via `--build-arg VITE_POSTHOG_KEY=phc_xxx`, Vite bakes it into the bundle
- Render automatically passes its env vars as Docker build args when the Dockerfile has matching ARG instructions

Do not add these ARGs to Stage 1 (deps) or Stage 3 (production) — they are only needed in the build stage where Vite runs.
