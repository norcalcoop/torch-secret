**RED phase -- write failing tests first.**

Create `client/src/crypto/__tests__/encoding.test.ts` with tests covering:

1. `uint8ArrayToBase64Url`:
   - Known input/output: `new Uint8Array([72, 101, 108, 108, 111])` ("Hello") -> verify output contains only URL-safe chars (no `+`, `/`, `=`)
   - Empty array -> empty string
   - 32-byte random-like array (simulating a key) -> verify no `+`, `/`, `=` in output
   - Binary data with bytes >127 (non-ASCII) -> produces valid base64url

2. `base64UrlToUint8Array`:
   - Round-trip: encode then decode produces identical Uint8Array
   - Known base64url string -> expected byte array

3. `uint8ArrayToBase64`:
   - Known input/output: `new Uint8Array([72, 101, 108, 108, 111])` -> `"SGVsbG8="`
   - Empty array -> empty string

4. `base64ToUint8Array`:
   - Round-trip: encode then decode produces identical Uint8Array
   - Known base64 string `"SGVsbG8="` -> `new Uint8Array([72, 101, 108, 108, 111])`

5. Round-trip property tests:
   - For a 256-bit (32-byte) random array: `base64UrlToUint8Array(uint8ArrayToBase64Url(bytes))` produces identical bytes
   - For a large array (1000 bytes): round-trip preserves all bytes
   - base64url output never contains `+`, `/`, or `=` characters (regex test `/[+/=]/`)

Run tests: `npx vitest run client/src/crypto/__tests__/encoding.test.ts` -- ALL tests MUST FAIL (module not yet implemented).

If vitest is not yet installed, first run `npm init -y && npm install -D vitest typescript` in the project root. Create a minimal `vitest.config.ts` at project root if needed.

**GREEN phase -- implement to pass.**

Create `client/src/crypto/encoding.ts` exporting four functions:

- `uint8ArrayToBase64Url(bytes: Uint8Array): string` -- convert bytes to binary string via loop using `String.fromCharCode(bytes[i])` (NOT spread operator to avoid stack overflow on large arrays), then `btoa()`, then replace `+` with `-`, `/` with `_`, strip trailing `=`
- `base64UrlToUint8Array(base64url: string): Uint8Array` -- reverse: replace `-` with `+`, `_` with `/`, pad with `=` to multiple of 4, `atob()`, then charCodeAt loop
- `uint8ArrayToBase64(bytes: Uint8Array): string` -- same as base64url but no character replacement, keep `=` padding
- `base64ToUint8Array(base64: string): Uint8Array` -- `atob()` then charCodeAt loop

Do NOT use `Buffer` (Node.js only, not available in browser). Do NOT use `Uint8Array.toBase64()` (not available in Node.js 24). Use only `btoa`/`atob` with manual conversion.

Run tests: `npx vitest run client/src/crypto/__tests__/encoding.test.ts` -- ALL tests MUST PASS.

**REFACTOR phase (if needed):** Clean up any duplication between base64 and base64url implementations (they share the byte-to-binary-string conversion).
