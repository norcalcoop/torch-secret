    **Step 3 — Import secrets into dev environment**
    Option A (CLI — fastest): Install Infisical CLI first (`brew install infisical/get-cli/infisical`), then:
    ```bash
    infisical login   # authenticate once
    cd /path/to/secureshare
    # Copy your real .env values temporarily, then import:
    infisical secrets set --env=dev --path=/ --file=.env
    ```
    Option B (Dashboard): Go to Project > Secrets > dev environment. Click "Add Secret" for each key. Use your real values from the working `.env` file.

    After import, verify the count: `infisical secrets --env=dev` should show ~23 secrets (all keys from `.env.example` plus `LOOPS_API_KEY` and `BETTER_AUTH_TRUSTED_ORIGINS`).

    Full list of keys to import (values from your working `.env`):
    - DATABASE_URL
    - PORT
    - LOG_LEVEL
    - NODE_ENV
    - FORCE_HTTPS (optional — can skip if unused)
    - REDIS_URL (optional — can skip if unused)
    - BETTER_AUTH_SECRET
    - BETTER_AUTH_URL
    - APP_URL
    - BETTER_AUTH_TRUSTED_ORIGINS
    - GOOGLE_CLIENT_ID (optional)
    - GOOGLE_CLIENT_SECRET (optional)
    - GITHUB_CLIENT_ID (optional)
    - GITHUB_CLIENT_SECRET (optional)
    - RESEND_API_KEY
    - RESEND_FROM_EMAIL
    - VITE_POSTHOG_KEY (optional)
    - VITE_POSTHOG_HOST (optional)
    - STRIPE_SECRET_KEY
    - STRIPE_WEBHOOK_SECRET
    - STRIPE_PRO_PRICE_ID
    - RESEND_AUDIENCE_ID
    - IP_HASH_SALT
    - LOOPS_API_KEY

    **Step 4 — Clone dev secrets to staging, override DATABASE_URL**
    In Infisical dashboard, go to the `staging` environment. Import/copy all secrets from `dev` using Project > Secrets > staging > "Import from environment: dev".
    Then override `DATABASE_URL` in staging to: `postgresql://secureshare:secureshare@db:5432/secureshare`
    (This is the Docker Compose internal hostname — required for `npm run staging:up` to work.)

    1. In Infisical: Organization Settings > Access Control > Machine Identities > Create Identity
    2. Name: `torch-secret-ci`
    3. Authentication method: OIDC Auth
    4. OIDC Discovery URL: `https://token.actions.githubusercontent.com`
    5. Issuer: `https://token.actions.githubusercontent.com`
    6. Subject (bound): `repo:norcalcoop/secureshare:*` (glob — matches all branches and tags)

    8. Click Save
    9. [SAVE THIS] Record the Identity ID as `INFISICAL_IDENTITY_ID`

    In the torch-secret project: Settings > Access Control > Identities > Add Identity
    Add `torch-secret-ci` with role: `developer` (or equivalent read access to `dev` environment).
