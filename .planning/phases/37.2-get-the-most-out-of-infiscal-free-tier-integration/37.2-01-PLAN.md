  <how-to-verify>
Complete the following steps at https://app.infisical.com and record the two values you'll need for Plan 02.

**Step 1 — Create the project:**
1. Sign in at https://app.infisical.com (create account if needed — free tier)
2. Click "New Project" → name it `torch-secret` → click Create
3. On the project settings page, copy the **Project ID** (also called workspaceId) — it looks like a UUID. You'll need this.

**Step 2 — Verify three environments exist:**
Infisical creates `dev`, `staging`, `prod` by default. Confirm these three exist in the project's Environments tab. Do NOT create a fourth environment (free tier limit is 3).

**Step 3 — Import secrets into the dev environment:**
- In the project, go to Secrets > dev environment
- Click "Add Secrets" → "Import from .env file"
- Upload or paste the contents of your local `.env` file (the real one with actual values, not `.env.example`)
- Confirm all secrets are imported. Expected count: ~23 keys. If the count is higher, remove deprecated/unused keys.

**Step 4 — Create OIDC machine identity for GitHub Actions:**
1. Go to Organization Settings > Access Control > Identities
2. Click "Create Identity" → name it `torch-secret-github-actions` → Role: Member
3. Under Authentication, select "OIDC Auth"
4. Configure bound claims:
   - OIDC Discovery URL: `https://token.actions.githubusercontent.com`
   - Issuer: `https://token.actions.githubusercontent.com`
   - Subject (bound): `repo:norcalcoop/secureshare:*` (glob matches all branches)
   - Audiences: `https://github.com/norcalcoop`
5. Save the identity. Copy the **Identity ID** — you'll need this.

**Step 5 — Add identity to the project:**
1. In the torch-secret project > Access Control > Identities
2. Add `torch-secret-github-actions` with `developer` or `member` role
3. Ensure it has read access to the `dev` environment

**Verification:**
- Project exists at https://app.infisical.com with three environments
- Secrets are visible in the dev environment (count approximately matches your .env)
- Machine identity `torch-secret-github-actions` appears in the project's access list
