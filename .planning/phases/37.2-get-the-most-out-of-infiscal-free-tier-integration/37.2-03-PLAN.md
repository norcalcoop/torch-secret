  <how-to-verify>
Complete both steps and then trigger a CI run to verify.

**Step 1 — Set INFISICAL_IDENTITY_ID as a GitHub Actions variable:**
1. Go to https://github.com/norcalcoop/secureshare/settings/variables/actions
2. Click "New repository variable"
3. Name: `INFISICAL_IDENTITY_ID`
4. Value: the identity-id from Plan 01 (the OIDC machine identity ID, not a secret)
5. Click Add variable

Note: This is a **variable** (not a secret) because the identity-id is not sensitive — it's effectively a public identifier. Infisical's security model relies on the OIDC token exchange, not the identity-id being secret.

**Step 2 — Configure Render Secret Sync in Infisical:**
1. In Render Account Settings > API Keys, create an API key named `infisical-sync`. Copy it.
2. In Infisical torch-secret project > Integrations > App Connections:
   - Click "Add Connection" → Render
   - Paste the Render API key → Save (connection named `render-prod`)
3. In Infisical > Integrations > Secret Syncs:
   - Click "Add Sync" → Render
   - Source: environment=`prod`, path=`/`
   - Destination: your Render service (select Service scope, not Environment Group)
   - Enable "Auto-Sync"
   - Click Create
4. Click "Sync Now" to do an immediate manual sync
5. In Render service > Environment: verify the secrets appear as env vars

**Step 3 — Populate prod environment in Infisical (if not yet done):**
If you haven't already, populate the `prod` environment in Infisical with your production values (live Stripe keys, production DATABASE_URL, etc.). The Render sync will push whatever is in the `prod` environment.

**Step 4 — Trigger CI and verify green:**
1. Push any commit (or manually trigger via GitHub Actions → Run workflow)
2. Observe the test job: the "Inject secrets from Infisical" step should succeed
3. "Run tests with coverage" should pass (328+ tests green)

**Verification:**
- GitHub Actions repo variable `INFISICAL_IDENTITY_ID` is set
- Infisical Render integration shows "Last synced: [recent timestamp]"
- Render service env vars include keys like `BETTER_AUTH_SECRET`, `STRIPE_SECRET_KEY`, etc.
- CI run is green (test job passes with Infisical injection)
