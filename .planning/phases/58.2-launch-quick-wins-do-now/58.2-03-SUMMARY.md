## Decisions Made

- **QR colors hardcoded:** `#000000/#ffffff` — never currentColor or CSS custom properties. Retro themes alter `--color-*` tokens dramatically; a QR code must have maximum contrast for reliable scanning regardless of active theme.
- **eslint-disable no-base-to-string:** The `@typescript-eslint/no-base-to-string` rule incorrectly flags `QRCode.toString(url, options)` because `toString` appears in TypeScript's Object prototype. Added `eslint-disable-next-line` with explanatory comment — this is a genuine false positive on a QR generation API.
- **mailto uses `<a>` not `<button>`:** Anchors support middle-click (open in mail client from new window), keyboard Enter activation on links, and correct `mailto:` protocol dispatch in all browsers. Styled identically to secondary buttons.
- **encodeURIComponent not encodeURI:** `encodeURI` does not encode `&` and `=`, which breaks the mailto query string parameters. `encodeURIComponent` also encodes `#` as `%23` — email clients decode this correctly back to `#` when the user clicks the link.
