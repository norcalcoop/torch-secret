<decisions>
## Implementation Decisions

### Mobile share sheet
- The Web Share API button already exists via `createShareButton` in `client/src/components/share-button.ts` and is rendered on the confirmation page
- This feature is considered already-shipped from prior phases; Claude should assess whether any enhancement (e.g., mobile layout prominence) is warranted
- Claude's Discretion: whether to swap button order on small screens so Share precedes Copy on mobile

### QR code
- Placement: toggle button in the button row — a "Show QR" button renders at the end of the [ Copy Link ] [ Share ] [ Email ] row; clicking expands an inline QR panel below the URL card
- Library: `qrcode` npm package, SVG output (`QRCode.toString(url, { type: 'svg' })`) — never innerHTML; use DOMParser to parse the SVG string, then appendChild
- Size and styling: compact 200×200px QR inside a centered card panel with caption "Scan to open on your phone"
- Theme colors: Claude's Discretion — keep QR modules dark-on-white regardless of retro theme for maximum scanner compatibility; the card container may use theme border/bg tokens

### Clipboard auto-clear countdown
- Duration: 60 seconds after each copy action
- Visual treatment: countdown badge on the copy button label — after copying, button text becomes "Copied — clears in 60s" and counts down live; resets to "Copy Link" when clipboard clears
- Both buttons affected: the "Copy Link" button AND the "Copy Passphrase" button (when passphrase was used) both get the countdown
- Re-copy behavior: Claude's Discretion — reset the timer to 60s on re-copy (natural expectation; consistent with how password managers handle re-copy)
- Implementation approach: extend `createCopyButton` in `copy-button.ts` to accept an optional `autoClearMs` param rather than wrapping it externally; confirmation.ts passes `autoClearMs: 60_000`

### Intelligent expiry suggestion
- Signal: time of day + day of week (client-side, no server call, no content inspection)
  - Friday afternoon (after 3pm): suggest 7 days — "Suggested: 7 days — recipient may not check until Monday"
  - Business hours Mon–Thu (9am–5pm): suggest 1 hour — "Suggested: 1 hour — recipient is likely available now"
  - Evening (after 7pm) / weekends: suggest 24 hours
  - All other times: suggest 24 hours (default)
- Appearance: hint text rendered below the expiration dropdown, non-prescriptive — user still manually selects; the hint informs but does not auto-change the selection
- Scope: authenticated users only — anonymous users are already locked to 1h with an upsell note; adding a hint on a locked selector adds noise without utility
- Implementation: `createExpirationSelect` gains an optional `suggestion?: { value: string; reason: string }` parameter; `create.ts` computes the suggestion and passes it in

### Mailto button
- Placement: in the button row alongside Copy and Share — [ Copy Link ] [ Share ] [ Email ]
- Subject line: "Secure message for you"
- Body: `[URL]\n\nThis link can only be viewed once.\nOpen it in a browser to reveal the secret.`
- Never includes the passphrase — two-channel security guidance card already tells the user to send it separately; combining them in one email defeats the two-channel model
- Implementation: `<a href="mailto:?subject=...&body=...">` styled as a button (or `window.open('mailto:...')`); Mail icon from Lucide

### Claude's Discretion
- QR theme adaptation (keep dark-on-white for scan reliability)
- Re-copy timer reset behavior (reset to 60s)
- Mobile layout ordering of share sheet button
- Exact Lucide icon choices for QR toggle and Email buttons

</decisions>

<specifics>
## Specific Ideas

- QR code: "toggle button that expands an inline panel" pattern mirrors the existing collapsible protection panel and label field patterns in create.ts — reuse the same expand/collapse approach
- Clipboard auto-clear: the countdown label changes are analogous to the existing icon-swap behavior in `swapToCheckIcon` — extend rather than replace that pattern
- Mailto body uses `encodeURIComponent` on the full URL (including fragment) — the `#key` fragment must survive as-is for the recipient to decrypt

- `client/src/pages/create.ts`: expiry suggestion logic lives here; compute suggestion before calling `createExpirationSelect`

### Established Patterns
- No innerHTML ever — SVG from qrcode library must be parsed via DOMParser then appended as DOM nodes
- DOM API for all element creation — `document.createElement`, `textContent`, `appendChild`
- `prefers-reduced-motion` guards on any animated countdown (use `window.matchMedia`)
- Button styling: primary = `bg-accent text-white`, secondary = `border border-border text-text-primary hover:bg-surface-raised` — mailto and QR toggle use secondary style
- Lucide icons via `createIcon(Icon, { size: 'sm' })` — consistent 16px icons in button labels

### Integration Points
- `confirmation.ts` → `copy-button.ts`: pass `autoClearMs: 60_000` when creating Copy Link and Copy Passphrase buttons
- `confirmation.ts` → new `qr-code-button.ts` (or inline in confirmation): QR toggle button + panel
- `confirmation.ts` → new `mailto-button.ts` (or inline): mailto anchor styled as button
- `create.ts` → `expiration-select.ts`: compute suggestion from `new Date()`, pass as `suggestion` param
- `package.json`: add `qrcode` dependency; add `@types/qrcode` as devDependency
