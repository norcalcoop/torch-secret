**Key implementation rules:**
1. Store `countdownInterval: ReturnType<typeof setInterval> | null = null` in the closure (above the click handler).
2. On each successful copy: if `autoClearMs` is set, call `clearInterval(countdownInterval)` first (handles re-copy), then start a new interval that counts down each second.
3. Check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` once at the start of each countdown. If reduced motion: skip label updates entirely but still clear clipboard at zero.
4. Clipboard clear: `void navigator.clipboard.writeText('').catch(() => {})` — empty string, never null/undefined.
5. Label reset: `labelSpan.textContent = defaultLabel` — use the closure variable, not a DOM query.
6. The `swapToCheckIcon` call still happens immediately on copy (icon swap is not a countdown).
7. The `labelSpan` reference already exists in the closure — mutate it directly.

**Interval pattern to implement:**
```typescript
// Inside createCopyButton closure, after declaring labelSpan and defaultLabel:
let countdownInterval: ReturnType<typeof setInterval> | null = null;

// Inside the async click handler, after successful clipboard.writeText(text):
if (options?.autoClearMs) {
  if (countdownInterval !== null) clearInterval(countdownInterval);
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  let remaining = Math.floor(options.autoClearMs / 1_000);
  if (!prefersReduced) {
    labelSpan.textContent = `Copied \u2014 clears in ${remaining}s`;
  }
  countdownInterval = setInterval(() => {
    remaining -= 1;
    if (remaining <= 0) {
      clearInterval(countdownInterval!);
      countdownInterval = null;
      void navigator.clipboard.writeText('').catch(() => {});
      labelSpan.textContent = defaultLabel;
    } else if (!prefersReduced) {
      labelSpan.textContent = `Copied \u2014 clears in ${remaining}s`;
    }
  }, 1_000);
}
```

Also add the same countdown start after the `fallbackCopy` path succeeds (the second try/catch branch). The fallback path does not call `navigator.clipboard.writeText` for the copy itself, so only clear on countdown expiry.
