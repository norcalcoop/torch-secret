## Summary

Phase 58.2 adds five high-value, self-contained client-side UX improvements to the confirmation page and create page. No API changes, no server changes, no new routes. All work is pure TypeScript + DOM in the existing Vite 7.x / Tailwind CSS 4.x client. The highest implementation risk is the `qrcode` npm package's CJS-only distribution, which requires a Vite `optimizeDeps` configuration to prevent a "require is not defined" build error. Every other feature involves straightforward extensions of existing component APIs.

The four new features can be grouped into two delivery areas: (1) confirmation page additions (QR code panel, clipboard auto-clear countdown, mailto button ‚Äî all within `renderConfirmationPage`) and (2) create page addition (intelligent expiry suggestion hint text in `createExpirationSelect`). The share button is already shipped and only warrants a mobile ordering assessment.

**Primary recommendation:** Write a dedicated `qr-code-panel.ts` component and a `mailto-button.ts` component rather than cramming all new logic into `confirmation.ts`. Extend `copy-button.ts` and `expiration-select.ts` as specified. All five plans should be independently merge-safe.

### Pattern 1: Extending `createCopyButton` with `autoClearMs`

**What:** Add an optional `autoClearMs` parameter. When set, after a successful copy the button label counts down (`Copied ‚Äî clears in 60s`, `...59s`, ...), clears the clipboard at zero, and resets the label to the original.

**Critical details:**
- The countdown interval MUST be stored in a closure variable and cleared on re-copy (re-copy resets to 60s, which means clearing the old interval before starting a new one).
- Clipboard clear: `await navigator.clipboard.writeText('')` ‚Äî same path as the original copy, so fallback also applies.
- `prefers-reduced-motion`: suppress the live countdown label updates if the user has requested reduced motion; still clear the clipboard at 60s, just do not animate the number live.
- The `labelSpan` reference already exists inside `createCopyButton`'s closure ‚Äî the countdown can mutate `labelSpan.textContent` directly without DOM search.

**Interval pattern:**
```typescript
// Source: derived from copy-button.ts closure structure (labelSpan already in scope)
let countdownInterval: ReturnType<typeof setInterval> | null = null;

function startCountdown(seconds: number): void {
  if (countdownInterval !== null) clearInterval(countdownInterval);
  let remaining = seconds;
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  if (!prefersReduced) {
    labelSpan.textContent = `Copied \u2014 clears in ${remaining}s`;
  }
  countdownInterval = setInterval(() => {
    remaining -= 1;
    if (remaining <= 0) {
      clearInterval(countdownInterval!);
      countdownInterval = null;
      void navigator.clipboard.writeText('').catch(() => { /* silent */ });
      labelSpan.textContent = defaultLabel;
    } else if (!prefersReduced) {
      labelSpan.textContent = `Copied \u2014 clears in ${remaining}s`;
    }
  }, 1_000);
}
```

### Pattern 2: `qr-code-panel.ts` ‚Äî Toggle + Inline Panel

**What:** A factory function that returns a button (the toggle) and a panel element (hidden by default). The panel holds the QR SVG + caption. Caller appends both to the button row and URL card area respectively.

**DOMParser SVG injection pattern (safe DOM insertion rule):**
```typescript
// Source: qrcode npm README + project convention (DOM API only, no unsafe string assignment)
import QRCode from 'qrcode';

async function renderQrSvg(url: string, container: HTMLElement): Promise<void> {
  const svgString = await QRCode.toString(url, {
    type: 'svg',
    errorCorrectionLevel: 'M',
    margin: 2,
    color: { dark: '#000000', light: '#ffffff' },  // dark-on-white regardless of theme
  });
  const doc = new DOMParser().parseFromString(svgString, 'image/svg+xml');
  const svgEl = doc.documentElement;
  svgEl.setAttribute('width', '200');
  svgEl.setAttribute('height', '200');
  svgEl.setAttribute('aria-hidden', 'true');
  // importNode adopts the element into the current document before appending
  const imported = document.importNode(svgEl, true);
  container.appendChild(imported);
}
```

**Toggle behavior:** Single click toggles `panel.hidden`. On first expand, call `renderQrSvg` (lazy ‚Äî do not generate on page load). Cache rendered state so subsequent toggles do not re-generate.

**Lucide icon choice:** `QrCode` (named export from `lucide` v0.575). Import as `import { QrCode } from 'lucide'`. Confirmed available at lucide.dev.

### Pattern 3: `mailto-button.ts` ‚Äî Anchor Styled as Button

**What:** An `<a>` element with `href="mailto:?subject=...&body=..."` styled with the secondary button class. All values are `encodeURIComponent`-encoded. The `#key` URL fragment must be present in the body ‚Äî `encodeURIComponent` encodes `#` as `%23`, which email clients decode correctly before opening the link in the browser.

**Fragment preservation note (MEDIUM confidence):** `encodeURIComponent` encodes `#` to `%23`. Email clients pass the full decoded URL to the browser when the recipient clicks. The browser parses the fragment normally. Do NOT strip the fragment.

**Lucide icon choice:** `Mail` (named export from `lucide` v0.575). Confirmed available at lucide.dev.

### Pattern 4: `createExpirationSelect` ‚Äî Suggestion Parameter

**What:** Add `suggestion?: { value: string; reason: string }` as a third optional parameter. When provided and the user is authenticated, render a `<p>` hint element below the combobox container with classes `text-xs text-text-muted mt-1`.

**Hint text format:** `"Suggested: {value label} ‚Äî {reason}"`

**Scoping:** The `if (!isAuthenticated)` branch returns early without rendering a hint ‚Äî anonymous locked display already has an upsell note.

**Return shape concern:** The current `ExpirationSelectResult.element` IS the combobox container. Adding a hint paragraph means wrapping `container` + `hint` in an outer div and returning that as `element`. Search all callers of `createExpirationSelect` first ‚Äî currently only one caller exists (`create.ts`), so the wrapper approach is safe.

**Day/time heuristics in `create.ts`:**
```typescript
// Source: CONTEXT.md decision; computed from browser Date API
function getExpirySuggestion(): { value: string; reason: string } | undefined {
  const now = new Date();
  const day = now.getDay();    // 0 = Sunday, 5 = Friday, 6 = Saturday
  const hour = now.getHours(); // 0-23

  const isWeekend = day === 0 || day === 6;
  const isFridayAfternoon = day === 5 && hour >= 15;
  const isBusinessHours = !isWeekend && !isFridayAfternoon && hour >= 9 && hour < 17;
  const isEvening = hour >= 19;

  if (isFridayAfternoon) {
    return { value: '7d', reason: 'recipient may not check until Monday' };
  }
  if (isBusinessHours) {
    return { value: '1h', reason: 'recipient is likely available now' };
  }
  if (isEvening || isWeekend) {
    return { value: '24h', reason: 'evening or weekend \u2014 try 24 hours' };
  }
  return { value: '24h', reason: 'default' };
}
```

**Integration point in `create.ts`:** Compute `suggestion` before calling `createExpirationSelect(isAuthenticated, isPro)` ‚Äî pass as third arg only for authenticated users (guard with `isAuthenticated` check before calling `getExpirySuggestion`).

### Pattern 5: Button Row Layout ‚Äî Mobile Share Ordering

**Current order (confirmation.ts):** `[ Copy Link ] [ Share ]`
**New order:** `[ Copy Link ] [ Share ] [ Email ] [ Show QR ]`

**Mobile ordering (Claude's Discretion):** The current `div.flex.items-center.gap-3` wraps naturally on narrow viewports. Copy + Share appear on the first visual row; Email + QR on the second when wrapping. No DOM reordering or `sm:order-*` classes are needed. This wrapping is acceptable behavior.

### Anti-Patterns to Avoid

- **Unsafe DOM string assignment for QR SVG:** Never assign a string directly to any DOM property that parses HTML or SVG. Use `DOMParser` + `importNode` + `appendChild`.
- **Setting QR color to `currentColor`:** QR must remain dark-on-white regardless of retro theme. Hard-code `color.dark: '#000000'` and `color.light: '#ffffff'`.
- **Clearing clipboard with `writeText(null)` or `writeText(undefined)`:** These throw. Use `writeText('')` (empty string is a valid value).
- **Running countdown interval without clearing on re-copy:** Store the interval in the closure; call `clearInterval` before starting a new one.
- **Encoding the body with `encodeURI` instead of `encodeURIComponent`:** `encodeURI` does not encode `&` or `=`, which breaks the `mailto:` query string parsing.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| QR code generation | Custom SVG grid from scratch | `qrcode` npm package | Reed-Solomon error correction, quiet zone, version selection ‚Äî all non-trivial |
| SVG parsing from string | Regex scraping | `DOMParser` (browser built-in) | Safe, standards-compliant, no dependencies |
| Clipboard clear | OS-level clipboard flush | `navigator.clipboard.writeText('')` | Works in all secure-context browsers; empty string is a valid clipboard value |

**Key insight:** QR error correction alone involves polynomial arithmetic over Galois fields. The `qrcode` package handles all of it; a hand-rolled SVG grid would produce unreadable codes.

## Common Pitfalls

### Pitfall 1: `qrcode` package CJS/ESM in Vite

**What goes wrong:** `qrcode` v1.5.4 ships as CommonJS only. In a Vite 7.x project (native ESM), importing it can produce a `require is not defined` error at runtime in the browser bundle.

**Why it happens:** Vite pre-bundles (via esbuild) CJS dependencies to ESM during `optimizeDeps`. If the package is not included in `optimizeDeps.include`, the transformation may be skipped or incomplete.

**How to avoid:** Add `qrcode` to `vite.config.ts` `optimizeDeps.include`:
```typescript
// vite.config.ts addition
optimizeDeps: {
  include: ['qrcode'],
},
```

**Warning signs:** Build succeeds but browser console shows `require is not defined` or `module is not defined` when the QR panel is first expanded.

**Confidence:** MEDIUM ‚Äî confirmed CJS-only package; `optimizeDeps.include` is the standard Vite fix for this class of package. Verify during Wave 0 build test.

### Pitfall 2: `qrcode` TypeScript import style

**What goes wrong:** Named destructured imports from `qrcode` do not work. The package uses `module.exports`.

**How to avoid:** Use the default import:
```typescript
import QRCode from 'qrcode';
// correct: await QRCode.toString(url, options)
```
The `@types/qrcode` 1.5.6 types expose `QRCode.toString` on the default export. This pattern matches the README examples.

### Pitfall 3: Countdown interval leak on re-copy

**What goes wrong:** User copies, countdown starts (interval A). User copies again before interval A finishes. Both intervals run in parallel ‚Äî label flickers and clipboard may be cleared at two different times.

**How to avoid:** Store the interval reference in the `createCopyButton` closure. On each successful copy, call `clearInterval(countdownInterval)` before starting a new one.

### Pitfall 4: `encodeURIComponent` double-encoding

**What goes wrong:** Calling `encodeURIComponent` on an already-encoded string doubles the encoding (e.g., `%23` becomes `%2523`).

**How to avoid:** Encode only raw values. The `shareUrl` received by `confirmation.ts` is a plain URL string (not pre-encoded). Call `encodeURIComponent(shareUrl)` exactly once.

### Pitfall 5: `DOMParser` returns `XMLDocument`; cross-document nodes need `importNode`

**What goes wrong:** The SVG element from `DOMParser.parseFromString(svgString, 'image/svg+xml')` belongs to a different document. Calling `container.appendChild(svgEl)` may work in most browsers but is technically incorrect and can fail in some environments.

**How to avoid:** Use `document.importNode(doc.documentElement, true)` to adopt the node into the current document before appending.

### Pitfall 6: `ExpirationSelectResult.element` shape change breaks callers

**What goes wrong:** Wrapping `container + hint` in an outer div changes what `ExpirationSelectResult.element` points to. If any code does `expirationResult.element.querySelector('#expiration')` expecting to find the combobox trigger, it still works (nested querySelector). But if code checks `expirationResult.element.id === 'expiration'`, it breaks.

**How to avoid:** Check all callers. Currently `create.ts` only uses `expirationResult.element` to `appendChild` it into the form. The wrapper approach is safe.

## Open Questions

1. **`qrcode` CJS/Vite compatibility in production build**
   - What we know: CJS packages require `optimizeDeps.include` in Vite to transform to ESM. Works in dev; production needs verification.
   - What's unclear: Whether esbuild in Vite 7.x handles `qrcode`'s CJS exports without additional `commonjsOptions`.
   - Recommendation: Add `optimizeDeps: { include: ['qrcode'] }` in vite.config.ts as Wave 0. Run `npm run build:client` and open the built output in a browser to confirm no `require` errors before Wave 1.

2. **`prefers-reduced-motion` and countdown label updates**
   - What we know: Existing `swapToCheckIcon` already checks `window.matchMedia('(prefers-reduced-motion: reduce)')`.
   - What's unclear: Whether a live-updating countdown label (every 1s) is considered "motion" under WCAG 2.1 SC 2.3.3.
   - Recommendation: Under `prefers-reduced-motion`, skip the live countdown label (show only the original label); still clear the clipboard after 60s silently. This is the safe default.

3. **`ExpirationSelectResult` return shape with suggestion wrapper**
   - What we know: Currently only `create.ts` calls `createExpirationSelect` and uses `.element` only to `appendChild`.
   - What's unclear: Future callers or tests that inspect the element shape.
   - Recommendation: Search for all `createExpirationSelect` imports before wrapping. One caller confirmed; wrapper approach is safe.
