The dropdown has this DOM structure:
```
<div class="relative" id="theme-dropdown-root">
  <button id="theme-dropdown-btn" type="button" aria-expanded="false" aria-haspopup="true" aria-label="Change theme">
    <!-- Current Lucide icon: Sun/Moon/Monitor or a palette icon when retro active -->
  </button>
  <div id="theme-dropdown-panel" role="menu" aria-label="Theme selector" class="hidden ...">
    <!-- Section 1: Base modes -->
    <div role="group" aria-label="Base modes">
      <button role="menuitem" data-theme="light">☀ Light</button>
      <button role="menuitem" data-theme="dark">🌙 Dark</button>
      <button role="menuitem" data-theme="system">🖥 System</button>
    </div>
    <hr aria-hidden="true" />
    <!-- Section 2: Retro Pro themes -->
    <div role="group" aria-label="Retro Pro themes">
      <!-- One button per theme in THEMES -->
      <button role="menuitem" data-retro-theme="mario">
        <!-- Lock icon if not Pro -->
        Mario
      </button>
      ...
    </div>
  </div>
</div>
```

Implementation details:
- Panel opens/closes by toggling `hidden` on `#theme-dropdown-panel`; `aria-expanded` on button reflects state
- Close panel on: Escape key, click outside (document click listener), after selection
- Hover preview:
  - `mouseenter` on retro theme button → call `applyRetroColors(THEMES[id])` (no localStorage write) — add 50ms debounce to avoid flicker (only the forward apply is debounced; mouseleave reverts immediately)
  - `mouseleave` on retro theme button → if `getRetroTheme()` is non-null, re-apply `applyRetroColors(THEMES[getRetroTheme()!])`; else call `clearRetroColors()`
- Pro gate on click: fetch `/api/me`, check `user.subscriptionTier === 'pro'`
  - If Pro: call `setRetroTheme(id)` then close panel
  - If free: call `navigate('/pricing')` then close panel; do NOT call setRetroTheme
  - Cache the Pro status in a module-level variable to avoid repeated fetches (refresh on `retrothemechange` event)
- Lock icon: use `createIcon(Lock, { size: 'sm' })` from lucide — shown when user is free
- Active state: apply `aria-pressed="true"` or a highlight class on the currently active theme button
- Keyboard: arrow keys navigate menu items; Enter activates; Escape closes
- Panel position: `absolute top-full right-0 mt-1 z-50` — drops below toggle button, right-aligned
- Panel styling: glassmorphism consistent with existing `bg-surface/80 backdrop-blur-md border border-border rounded-xl shadow-lg` pattern; max-height with overflow-y-auto for long retro list
- Theme group labels: small uppercase muted label above each group (`Base Modes`, `Retro Pro — upgrade to unlock`)
- When retro theme is active, toggle button icon can be the Palette lucide icon (or leave as Monitor/Sun/Moon — use Palette from 'lucide')
