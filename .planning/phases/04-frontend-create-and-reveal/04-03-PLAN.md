    3. Create `client/src/pages/reveal.ts`:
       - Export async function `renderRevealPage(container: HTMLElement): Promise<void>`
       - SECURITY-CRITICAL flow (RETR-05, RETR-06):

       Step A - Extract and strip key (IMMEDIATELY, before any rendering):
         - `const key = window.location.hash.slice(1);` -- extract key from fragment
         - `history.replaceState(null, '', window.location.pathname + window.location.search);` -- strip fragment from URL bar
         - The key now exists only in the `key` variable in memory

       Step B - Extract secret ID from path:
         - Parse window.location.pathname to get the ID: `const segments = window.location.pathname.split('/'); const id = segments[segments.length - 1];`

       Step C - Validate inputs (client-side checks can show distinct errors since they don't reveal server state):
         - If !key (empty string): render error page with 'no_key' type, return immediately. Do NOT fetch from API.
         - If !id or id.length !== 21: render error page with 'not_available' type, return immediately.

       Step D - Render interstitial (RETR-05):
         - Clear container
         - Show centered content:
           - Lock/shield icon (simple SVG or Unicode)
           - Heading: "You've received a secret"
           - Subtext: "This secret can only be viewed once. Once revealed, it will be permanently destroyed."
           - Button: "Reveal Secret" with primary styling (bg-primary-600, text-white, px-8, py-3, rounded-lg, font-semibold, text-lg)
         - The button's click handler calls handleReveal(container, id, key)
         - CRITICAL: NO API call happens here. NO fetch. The page is inert until the user clicks.

       Step E - handleReveal function (called only on button click):
         - Replace interstitial content with loading spinner: createLoadingSpinner("Decrypting your secret...")
         - Try:
           a. `const { ciphertext } = await getSecret(id);` -- fetches from API (triggers atomic delete on server)
           b. `const plaintext = await decrypt(ciphertext, key);` -- decrypts client-side using Phase 1 crypto
           c. Render the revealed secret (Step F)
           d. Set key variable to null (best-effort memory cleanup -- the key was in a closure)
         - Catch:
           a. If error is ApiError with status 404 or 410: render error page 'not_available'
           b. If decryption error: render error page 'decrypt_failed'
           c. For any other error: render error page 'not_available' (don't leak error details)

       Step F - Render revealed secret:
         - Clear container
         - Heading: "Secret Revealed"
         - Notice: "This secret has been permanently destroyed from our servers."
         - Secret display area:
           - Use a `<pre>` element with classes: whitespace-pre-wrap, break-words, bg-white, border, border-gray-200, rounded-lg, p-4, text-sm, font-mono, max-h-96, overflow-y-auto
           - Set content via `pre.textContent = plaintext` -- NEVER innerHTML (XSS prevention, Pitfall 4)
         - Copy button: createCopyButton(() => plaintext, "Copy Secret") positioned below the secret
         - "Create a New Secret" link back to /

       - All DOM content created via createElement + textContent. Zero innerHTML usage.
       - Import decrypt from '../crypto' (uses barrel export)
       - Import getSecret, ApiError from '../api/client'
       - Import createCopyButton from '../components/copy-button'
       - Import createLoadingSpinner from '../components/loading-spinner'
       - Import renderErrorPage from './error' (for error states)
       - Import navigate from '../router' (for back links)
