<task type="auto">
  <name>Task 2: Create client router, API client, copy button component, app entry point, and wire Express production serving</name>
  <files>

  <action>
    1. Create `client/src/router.ts`:
       - Type `PageRenderer = (container: HTMLElement) => void | Promise<void>`
       - Export `navigate(path: string): void` -- calls history.pushState(null, '', path) then handleRoute()
       - Export `initRouter(): void` -- calls handleRoute() and registers window 'popstate' listener
       - Internal `handleRoute(): void`:
         - Gets path from window.location.pathname
         - Gets container from document.getElementById('app')!
         - Clears container children with while(container.firstChild) container.removeChild(container.firstChild)
         - Route matching:
           - if path === '/' -> dynamically import('../pages/create.js') and call renderCreatePage(container)
           - if path.startsWith('/secret/') -> dynamically import('../pages/reveal.js') and call renderRevealPage(container)
           - else -> dynamically import('../pages/error.js') and call renderErrorPage(container, 'not_found')
       - Use dynamic imports for code splitting -- each page is a separate chunk
       - Handle import errors gracefully (show generic error if chunk fails to load)

    2. Create `client/src/api/client.ts`:
       - Export class `ApiError extends Error` with `status: number` and `body: unknown` fields
       - Export async function `createSecret(ciphertext: string, expiresIn: '1h' | '24h' | '7d' | '30d'): Promise<CreateSecretResponse>`
         - POST to '/api/secrets' with JSON body { ciphertext, expiresIn }
         - On !res.ok: throw new ApiError(res.status, await res.json())
         - Return res.json() typed as CreateSecretResponse
       - Export async function `getSecret(id: string): Promise<SecretResponse>`
         - GET '/api/secrets/{id}'
         - On !res.ok: throw new ApiError(res.status, await res.json())
         - Return res.json() typed as SecretResponse
       - Import types from '../../../shared/types/api' (same relative path pattern as server code)
       - IMPORTANT: Never cache GET responses (each call atomically destroys the secret)

    3. Create `client/src/components/copy-button.ts`:
       - Export function `createCopyButton(getText: () => string, label?: string): HTMLButtonElement`
       - Creates a button element with Tailwind classes for styling (inline-flex, items-center, gap-2, px-4, py-2, rounded-lg, bg-primary-600, text-white, hover:bg-primary-700, transition-colors, cursor-pointer)
       - Button text defaults to label or "Copy"
       - On click: calls getText(), then navigator.clipboard.writeText(text)
         - On success: change button text to "Copied!", add text-success-500 class, setTimeout 2000ms to restore
         - On failure: fallback -- create hidden textarea, set value, select, document.execCommand('copy'), remove textarea
       - Returns the button element

    4. Create `client/src/app.ts`:
       - Import './styles.css' (Vite handles CSS imports)
       - Import { initRouter } from './router'
       - On DOMContentLoaded: call initRouter()
       - NOTE: Do NOT create placeholder page files yet. The router uses dynamic imports which will only be called when the route matches. The build won't fail because dynamic imports are lazy.

    5. Update `server/src/app.ts` for production static serving:
       - Add conditional production serving AFTER the API routes and BEFORE the error handler:
         - Import `readFileSync` from 'node:fs', `resolve` from 'node:path'
         - Import `express` static middleware (already available from express import)
         - In buildApp(), after `app.use('/api/secrets', createSecretsRouter())`:
           - Compute `clientDistPath = resolve(import.meta.dirname, '../../client/dist')`
           - Serve static assets: `app.use(express.static(clientDistPath, { index: false }))` -- index:false because we handle HTML ourselves
           - Read HTML template once: `const htmlTemplate = readFileSync(resolve(clientDistPath, 'index.html'), 'utf-8')`
           - SPA catch-all: `app.get('*', (req, res) => { ... })` that replaces '__CSP_NONCE__' with res.locals.cspNonce using replaceAll, sets Content-Type text/html, sends
         - Wrap the static/catch-all in a try-catch or existence check so the server still starts in dev mode when client/dist doesn't exist yet. Use `import { existsSync } from 'node:fs'` to check if clientDistPath exists before setting up static serving.
       - Keep the error handler as the LAST middleware (after the catch-all)

    6. Update `vitest.config.ts` to handle frontend tests with happy-dom:
       - Add environmentMatchGlobs to the test config: `environmentMatchGlobs: [['client/src/**/*.test.ts', 'happy-dom']]`
       - This makes client tests use happy-dom while server tests stay on node environment
       - Keep all other existing config unchanged
  </action>
