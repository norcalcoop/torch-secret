  <how-to-verify>
    **Prerequisites:**
    1. Start the app: `npm run dev:server` and `npm run dev:client`
    2. OR build and run production: `npm run build:client && npm run dev:server`

    **Test 1 — SEO-01: VS pages return H1 in initial HTTP response:**
    Run each of:
    ```
    curl -s http://localhost:3000/vs/onetimesecret | grep '<h1'
    curl -s http://localhost:3000/vs/pwpush | grep '<h1'
    curl -s http://localhost:3000/vs/privnote | grep '<h1'
    ```
    Expected: Each returns one line containing `<h1` with the page heading text.
    NOT expected: empty output (would mean SPA shell served instead).
    NOT expected: `__CSP_NONCE__` in output (would mean SPA template served instead of SSR).

    **Test 2 — SEO-02: Alternatives pages return H1 in initial HTTP response:**
    Run each of:
    ```
    curl -s http://localhost:3000/alternatives/onetimesecret | grep '<h1'
    curl -s http://localhost:3000/alternatives/pwpush | grep '<h1'
    curl -s http://localhost:3000/alternatives/privnote | grep '<h1'
    ```
    Expected: Each returns one line containing `<h1` with the page heading text.

    **Test 3 — SEO-03: Use-case hub shows card grid:**
    - Open http://localhost:3000/use/ in a browser (or run `curl -s http://localhost:3000/use/ | grep '<h1'`)
    - Expected: Page loads with an H1 heading and a grid of cards linking to all 8 use-case pages
    - Verify at least one card link to `/use/share-api-keys` is visible

    **Test 4 — SEO-04: Individual use-case pages are substantive:**
    - Open http://localhost:3000/use/share-api-keys in a browser
    - Expected: Page has H1 "How to Share API Keys Securely", substantive content (problem, steps, FAQ), CTA button
    - Run: `curl -s http://localhost:3000/use/send-password-without-email | grep '<h1'`
    - Expected: Returns H1 heading line

    **Test 5 — SEO-05: JSON-LD structured data in <head>:**
    Run:
    ```
    curl -s http://localhost:3000/vs/onetimesecret | grep -o '"@type":"FAQPage"'
    curl -s http://localhost:3000/alternatives/onetimesecret | grep -o '"@type":"FAQPage"'
    curl -s http://localhost:3000/use/share-api-keys | grep -o '"@type":"HowTo"'
    curl -s http://localhost:3000/use/share-api-keys | grep -o '"@type":"FAQPage"'
    ```
    Expected: Each command returns the matched string (not empty).

    **Test 6 — SEO-06a: No noindex header on SEO pages:**
    Run:
    ```
    curl -sI http://localhost:3000/vs/onetimesecret | grep -i x-robots
    curl -sI http://localhost:3000/alternatives/onetimesecret | grep -i x-robots
    curl -sI http://localhost:3000/use/share-api-keys | grep -i x-robots
    ```
    Expected: No output (no X-Robots-Tag header returned).
    If output appears: it must NOT contain "noindex".

    **Test 7 — SEO-06b: Sitemap.xml contains all SEO routes:**
    Run:
    ```
    grep -c '<url>' client/public/sitemap.xml
    grep '/vs/onetimesecret\|/alternatives/onetimesecret\|/use/' client/public/sitemap.xml
    ```
    Expected: `grep -c` returns 17 or more. The grep returns matches for all three sample URLs.

    **Test 8 — Visual design check (browser):**
    - Open http://localhost:3000/vs/onetimesecret in a browser
    - Expected: Page visually matches SPA design (same fonts, same colors, glassmorphism card for hero section, nav with Home/Create/Pricing/Dashboard links, footer)
    - Toggle browser to dark mode (or use system dark mode)
    - Expected: Page switches to dark colors without JS (CSS-only prefers-color-scheme media query)
    - Confirm no unstyled flash (page loads styled immediately)
  </how-to-verify>
