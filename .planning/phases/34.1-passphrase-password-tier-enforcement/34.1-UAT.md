## Tests

### 1. Anonymous User - Protection Panel Lock Indicators
expected: On the Create page as a logged-out (anonymous) user, the protection panel shows tier indicators. The passphrase tab is accessible (Free tier), while the generate/custom password tabs are visually locked with a "Pro" badge. Clicking a locked tab does NOT activate it (tab content does not change).
result: [pass] Browser verified: passphrase=Free badge (aria-disabled), generate/custom=Pro badge (aria-disabled), no-protection=active by default, clicking locked tab does not change aria-selected.

### 2. Locked Tab Popover CTA
expected: Clicking a locked tab (generate or custom password) while anonymous/free shows a popover with a CTA link — clicking it navigates to /register (for anonymous) or /pricing (for free users). The popover appears near the tab and has a link/button.
result: [pass] Browser verified: passphrase tab click shows #lock-popover-passphrase with "Sign up free" CTA href=/register; generate tab click shows #lock-popover-generate with "Upgrade" CTA href=/pricing.

### 3. Arrow Key Skips Locked Tabs
expected: With focus in the protection panel tab list, pressing the arrow keys skips over locked (Pro-only) tabs and moves focus only to unlocked tabs.
result: [pass] Browser verified: free user ArrowRight from "No protection" jumps directly to "Passphrase" (aria-selected=true), skipping Pro-locked generate and custom tabs.

### 4. Free User - Passphrase Unlocked, Password Locked
expected: Logged in as a free-tier user, the passphrase tab is fully unlocked and activatable. The generate/custom password tabs still show "Pro" lock indicators and cannot be activated.
result: [pass] Browser verified: passphrase tab has no aria-disabled and activates on click; generate and custom tabs retain aria-disabled=true with "Pro" badge.

### 5. Pro User - All Tabs Unlocked
expected: Logged in as a Pro user, all protection type tabs (passphrase, generate, custom password) are fully unlocked with no lock indicators. Clicking each tab activates it and shows the appropriate content/controls.
result: pass
note: Earlier intermediate automated run failed due to Playwright cookie persistence between free→Pro login (no `close` step). Final TESTS.yaml spec (with `close` + separate state files) confirmed 10/10 PASS. Implementation verified correct: getLockLevel() returns 'none' for all tabs when isPro=true.

### 6. Create Secret with Protection Type
expected: As a logged-in user (free or Pro), selecting the passphrase tab and creating a secret succeeds. The secret is created and the confirmation/share URL is shown — confirming protection_type is correctly sent to the server.
result: [pass] Browser verified: free user selects Passphrase tab, fills secret, submits — "Your Secure Link is Ready" heading shown and /secret/ URL visible in confirmation.
