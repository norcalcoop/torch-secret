<task type="checkpoint:human-action">
  <name>Task 1: Create dedicated Resend API key for Gmail SMTP relay (GMAI-01)</name>
  <action>
    Log in to resend.com and navigate to API Keys (resend.com/api-keys).

    1. Click "Create API Key"
    2. Fill in the creation form:
       - Name: Gmail SMTP Relay
       - Permission: Sending access (not Full access)
       - Domain Access: torchsecret.com (not "All domains")
    3. Click "Add" to create the key
    4. CRITICAL — Copy the key value immediately. The key is shown in plaintext only once.
       - Paste it into your password manager (entry name: "Resend API Key — Gmail SMTP Relay")
       - Do NOT close or navigate away from this dialog until the key is saved
    5. Confirm: the API Keys page shows the new entry "Gmail SMTP Relay" scoped to torchsecret.com

    Notes:
    - Permission level: "Sending access" is sufficient — do not grant Full access
    - Domain restriction limits blast radius if the key is ever leaked
    - If you close the dialog without copying the key, it cannot be retrieved — you must delete and recreate
  </action>
  <verify>Resend dashboard → API Keys shows "Gmail SMTP Relay" entry scoped to torchsecret.com. Key value saved in password manager.</verify>

<task type="checkpoint:human-action">
  <name>Task 2: Add all 7 Gmail Send mail as aliases in one session (GMAI-02)</name>
  <action>
    Open Gmail (mail.google.com) logged in as torch.secrets@gmail.com.

    Navigate to: Settings (gear icon) → See all settings → Accounts and Import tab → Send mail as section.

    For EACH of the 7 aliases below, click "Add another email address" and complete both dialog screens:

    Screen 1 (identity):
    - Name: Torch Secret
    - Email address: [alias]@torchsecret.com
    - Leave "Treat as an alias" checked
    - Click "Next Step"

    Screen 2 (SMTP):
    - SMTP Server: smtp.resend.com
    - Port: 465
    - Username: resend  ← literal string "resend", NOT the email address, NOT the API key
    - Password: [Resend API key from Task 1 — paste from password manager]
    - Select: SSL  ← must be SSL, not TLS/STARTTLS (port 465 = implicit SSL)
    - Click "Add Account"

    Aliases to add in order:
    1. hello@torchsecret.com
    2. contact@torchsecret.com
    3. admin@torchsecret.com
    4. info@torchsecret.com
    5. support@torchsecret.com
    6. security@torchsecret.com
    7. privacy@torchsecret.com

    IMPORTANT: Add all 7 before verifying any. After clicking "Add Account" for each alias, Gmail sends a verification email and returns to the Settings page. Do not click verification links yet — proceed to the next alias immediately.

    After all 7 are added:
    - Gmail Settings → Accounts and Import → Send mail as should show 7 new entries, each with a "verify" link or "pending" indicator
    - Each entry must show "Send through smtp.resend.com" (NOT "Send through Gmail")

    Troubleshooting:
    - "Username and password not accepted" → Username must be the literal string "resend" (not email address, not API key)
    - "Failed to connect" → Confirm port is 465 AND SSL is selected (not TLS/STARTTLS)
    - Entry shows "Send through Gmail" → SMTP was not saved; remove the alias and re-add with correct SMTP credentials
  </action>
