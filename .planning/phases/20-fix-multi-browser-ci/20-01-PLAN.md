  <action>
Make three targeted changes to the e2e job in `.github/workflows/ci.yml`. These are the only changes needed. Do not modify anything else.

**Change 1 — Install Playwright browsers (cache miss branch):**
Current (line 137):
```
run: npx playwright install --with-deps chromium
```
Change to:
```
run: npx playwright install --with-deps
```
Removing `chromium` causes `playwright install` to install all browsers (Chromium, Firefox, WebKit) plus all required OS-level system dependencies for each.

**Change 2 — Install Playwright system deps (cache hit branch):**
Current (line 141):
```
run: npx playwright install-deps chromium
```
Change to:
```
run: npx playwright install-deps
```
When binaries are restored from cache, system dependencies are not cached and must be installed. Removing `chromium` installs system deps for all installed browsers (Firefox and WebKit need different system libraries; `chromium`-specific install would silently leave them uninstalled).

**Change 3 — Run E2E tests without project filter:**
Current (line 147):
```
run: npx playwright test --config e2e/playwright.config.ts --project=chromium
```
Change to:
```
run: npx playwright test --config e2e/playwright.config.ts
```
Removing `--project=chromium` lets Playwright defer to `e2e/playwright.config.ts`, which already defines three projects (chromium, firefox, webkit). All three will execute.

**Do NOT change:**
- The cache key (`playwright-${{ runner.os }}-${{ hashFiles('package-lock.json') }}`). It is already correct for multi-browser use — it keys on `@playwright/test` version, which determines which binaries are valid regardless of browser count.
- The cache path (`~/.cache/ms-playwright`). This is where Playwright stores all browsers.
- Anything else in the workflow file.
