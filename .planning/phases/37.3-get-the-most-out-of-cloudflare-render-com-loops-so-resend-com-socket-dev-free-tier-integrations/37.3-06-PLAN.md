**Step 2: Cloudflare Cache Rules**
Navigate to: Cloudflare Dashboard → torchsecret.com → Caching → Cache Rules → Create Rule

Create two rules:

**Rule 1 — Static assets (permanent cache):**
- Rule name: `Static assets — Vite hashed`
- When: URI Path matches the wildcard `/assets/*`
- Cache eligibility: Eligible for cache
- Edge TTL: Override (set to maximum available — 1 year / 365 days if allowed, otherwise the maximum value the UI permits)
- Browser TTL: Override (same maximum value)
- Notes: Vite content-hashes filenames — cache busting is by filename change, not expiry. If 1 year TTL is not available on the free tier, use whatever maximum the dashboard allows and note the actual TTL used here.

**Rule 2 — HTML entry point (bypass or short cache):**
- Rule name: `SPA entry point — no cache`
- When: URI Path does not contain `/assets/` AND URI Path does not contain `/api/`
  (This matches /, /create, /reveal/*, /pricing, etc.)
- Cache eligibility: Bypass cache
  OR if Bypass is not available on free tier: Cache with Edge TTL Override of 5 minutes / Browser TTL of 0 (no browser cache)

**Step 3: Bot Fight Mode**
Navigate to: Cloudflare Dashboard → torchsecret.com → Security → Bots
- Enable "Bot Fight Mode" (toggle to On)
- Verify DDoS protection status shows as "Active" (it is on by default for proxied domains)

**Step 4: Cloudflare Web Analytics**
Navigate to: Cloudflare Dashboard → torchsecret.com → Analytics → Web Analytics
- If not already enabled: click "Enable Web Analytics"
- For proxied domains, no JS snippet is needed — Cloudflare collects data automatically
- Verify the analytics dashboard shows torchsecret.com as an active property
