<task type="auto">
  <name>Task 2: Refactor updatePageMeta() to PageMeta object and update all call sites</name>
  <files>client/src/router.ts, client/src/pages/confirmation.ts</files>
  <action>
1. **Refactor `client/src/router.ts`**:

   a. Add a `PageMeta` interface at the top of the file (exported):
   ```typescript
   export interface PageMeta {
     title: string;
     description: string;
     canonical?: string;   // defaults to window.location.origin + window.location.pathname
     noindex?: boolean;    // when true, adds <meta name="robots" content="noindex, nofollow">
   }
   ```

   b. Rewrite `updatePageMeta()` to accept `PageMeta` instead of `string`. The function must manage ALL meta elements as the single source of truth -- no stale tags persist. Implementation:

   - Set `document.title = meta.title + ' - SecureShare'` (existing behavior)
   - Create-or-update `<meta name="description">` with `meta.description`
   - Create-or-update OR remove `<link rel="canonical">`:
     - If `meta.noindex` is true: REMOVE the canonical element (noindex pages should not have canonical per research Pitfall 4)
     - Otherwise: set `href` to `meta.canonical ?? (window.location.origin + window.location.pathname)`
   - Create-or-update OR remove `<meta name="robots">`:
     - If `meta.noindex` is true: set content to `'noindex, nofollow'`
     - Otherwise: remove the robots meta element (let search engine default to index)
   - Call `updateOgTags(meta.noindex ?? false)` to swap OG values
   - Preserve the existing aria-live announcer behavior (clear then set via requestAnimationFrame)

   c. Add `updateOgTags(isNoindex: boolean)` as a private helper function:
   - Query for `meta[property="og:title"]`, `meta[property="og:description"]`, `meta[property="og:url"]`, `meta[name="twitter:title"]`, `meta[name="twitter:description"]`
   - If `isNoindex`:
     - Set og:title and twitter:title to `'SecureShare'`
     - Set og:description and twitter:description to `'Zero-knowledge secret sharing'`
     - Set og:url to `window.location.origin + '/'`
   - If NOT isNoindex (homepage):
     - Restore og:title and twitter:title to `'SecureShare - Zero-Knowledge Secret Sharing'`
     - Restore og:description and twitter:description to `'End-to-end encrypted. One-time view. No accounts.'`
     - Set og:url to `window.location.origin + '/'`

   d. Update the three `handleRoute()` call sites from string to PageMeta object:
   - `'/'` route: `updatePageMeta({ title: 'Share a Secret', description: 'Share secrets securely with zero-knowledge encryption. One-time view, no accounts, end-to-end encrypted in your browser.' })`
   - `/secret/*` route: `updatePageMeta({ title: "You've Received a Secret", description: 'Zero-knowledge secret sharing', noindex: true })`
   - 404/else route: `updatePageMeta({ title: 'Page Not Found', description: 'Zero-knowledge secret sharing', noindex: true })`

2. **Update `client/src/pages/confirmation.ts`**:
   - Change the `updatePageMeta('Your Secure Link is Ready')` call to:
     ```typescript
     updatePageMeta({
       title: 'Your Secure Link is Ready',
       description: 'Share secrets securely with zero-knowledge encryption. One-time view, no accounts, end-to-end encrypted in your browser.',
     })
     ```
   - The confirmation page does NOT set `noindex` because the URL is still `/` (it is state-based). It preserves the homepage description and canonical.
   - Update the import to include `PageMeta` if needed for type checking, though since `updatePageMeta` accepts `PageMeta` the type will be inferred from the object literal.

3. **Verify no missed call sites**: Run `npx tsc --noEmit` -- TypeScript will error on any call site still passing a string since the parameter type is now `PageMeta` (an interface, not a union with string).
