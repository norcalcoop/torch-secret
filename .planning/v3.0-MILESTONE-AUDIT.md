### Risk Flags (Tech Debt, Not Blockers)

**Risk 1 — Playwright webServer 30s timeout may be tight in CI** (affects CICD-02, TEST-02..04, TEST-06..07)

`e2e/playwright.config.ts` `webServer.command` runs `npm run build:client && npm run dev:server` with `timeout: 30_000`. Playwright starts polling `http://localhost:3000/api/health` immediately when the command launches, but the health endpoint is unavailable until both `vite build` (15–40s on cold CI runner) and `dev:server` startup (~2s) complete. On a slow runner this may exceed 30s, causing intermittent E2E failures.

**Fix:** Add a dedicated `npm run build:client` step to the CI `e2e` job before `npx playwright test`. This pre-builds the client assets so the webServer command only needs to run `dev:server`, completing in ~2s within the 30s budget.

**Risk 2 — Codecov token requires manual GitHub secret** (affects REPO-01)

`codecov/codecov-action@v5` uses `fail_ci_if_error: false` so CI passes without the token. The Codecov coverage badge shows "unknown" until `CODECOV_TOKEN` is added to GitHub repository secrets.

**Risk 3 — `docker run <image>` without command override skips migrations**

`Dockerfile` `CMD` launches the server directly. `docker-compose.yml` and `render.yaml` both override with the `migrate && server` chain. The README only shows `docker compose up`. Bare `docker run` is an unsupported path but could confuse users who try it.
