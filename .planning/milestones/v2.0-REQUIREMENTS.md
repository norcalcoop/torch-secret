# Requirements: SecureShare

**Defined:** 2026-02-15
**Core Value:** Users can share sensitive information once, securely, without accounts or complexity -- the secret is encrypted in the browser, viewable only once, then permanently destroyed.

## v2.0 Requirements

Requirements for the Developer-Grade UI & SEO milestone. Each maps to roadmap phases.

### Theme & Design System

- [x] **THEME-01**: App displays with dark terminal-inspired color scheme as default (deep navy-black backgrounds, light text) — Phase 9
- [x] **THEME-02**: User can toggle between dark, light, and system color scheme preferences — Phase 13
- [x] **THEME-03**: Theme preference persists across page refreshes and navigation via localStorage — Phase 13
- [x] **THEME-04**: All colors use semantic CSS custom property tokens (surface, border, text) switchable by theme — Phase 9
- [x] **THEME-05**: Primary content areas use glassmorphism card surfaces (backdrop-blur, translucent backgrounds) — Phase 13
- [x] **THEME-06**: Page background displays subtle dot-grid pattern for visual texture — Phase 11
- [x] **THEME-07**: Browser chrome (scrollbars, form controls) adapts to dark theme via color-scheme — Phase 9

### Typography

- [x] **TYPO-01**: Headings and code contexts display in JetBrains Mono (self-hosted, no external CDN) — Phase 9
- [x] **TYPO-02**: Body text uses system sans-serif for readability — Phase 9
- [x] **TYPO-03**: Revealed secret displays in terminal code-block style (dark bg, green-tinted monospace, header bar with copy button) — Phase 12

### Icons

- [x] **ICON-01**: All emoji icons replaced with Lucide SVG icons using CSP-safe presentation attributes — Phase 11
- [x] **ICON-02**: Icon utility module provides consistent defaults (stroke-width, aria-hidden) via tree-shakable imports — Phase 9
- [x] **ICON-03**: Theme toggle uses Sun/Moon/Monitor icons for dark/light/system states — Phase 13

### Layout

- [x] **LAYOUT-01**: Persistent header displays shield icon + "SecureShare" wordmark on all pages — Phase 11
- [x] **LAYOUT-02**: Non-create pages show a subtle "Create" navigation link in header — Phase 11
- [x] **LAYOUT-03**: Persistent footer displays trust signals ("Zero-knowledge encryption" / "AES-256-GCM" / "Open Source") — Phase 11
- [x] **LAYOUT-04**: Header and footer persist across SPA route changes (rendered outside #app) — Phase 11

### UI Enhancements

- [x] **UI-01**: Confirmation page URL display combines input + copy button inline (single prominent block) — Phase 12
- [x] **UI-02**: Textarea shows "Encrypted in your browser" indicator with lock icon when content is present — Phase 12
- [x] **UI-03**: "How It Works" section uses SVG icons (browser/shield, server/lock, flame/destroy) instead of numbered circles — Phase 12
- [x] **UI-04**: Revealed secret destruction confirmation displays as prominent green success badge with checkmark icon — Phase 12
- [x] **UI-05**: Create page includes "Why Trust Us?" section with zero-knowledge, open source, no accounts, AES-256 points — Phase 12
- [x] **UI-06**: Page transitions use fade-in-up animation (200ms ease-out) — Phase 13
- [x] **UI-07**: Buttons have hover/active micro-interactions (scale, color shift) — Phase 13
- [x] **UI-08**: Copy button shows icon swap animation (Copy to Check) on success — Phase 13
- [x] **UI-09**: All animations respect prefers-reduced-motion via motion-safe / motion-reduce variants — Phase 13

### SEO

- [x] **SEO-01**: Homepage has meta description, og:title, og:description, og:image, twitter:card tags — Phase 14
- [x] **SEO-02**: Favicon suite serves favicon.svg (dark mode adaptive), favicon.ico, and apple-touch-icon.png — Phase 10
- [x] **SEO-03**: robots.txt serves from client/public/ with proper directives (allow /, disallow /api/) — Phase 10
- [x] **SEO-04**: sitemap.xml lists only the homepage URL — Phase 10
- [x] **SEO-05**: Secret routes (/secret/*) get noindex, nofollow via dynamic meta tag and X-Robots-Tag header — Phase 14
- [x] **SEO-06**: Secret routes serve minimal/generic OG tags (no indication that a secret exists at that URL) — Phase 14
- [x] **SEO-07**: JSON-LD structured data block with WebApplication schema on homepage — Phase 14
- [x] **SEO-08**: Web manifest (site.webmanifest) with app name, icons, and dark theme colors — Phase 10
- [x] **SEO-09**: Noscript fallback content inside #app for crawlers that don't execute JS — Phase 10
- [x] **SEO-10**: SPA router updatePageMeta() refactored to set title, description, canonical, and noindex per route — Phase 14

## Out of Scope

| Feature | Reason |
|---------|--------|
| Full PWA with service worker | Caching secrets is a security risk; offline mode meaningless for network-dependent tool |
| SSR/pre-rendering for SEO | Only 1 indexable page; architecture change for near-zero ROI |
| CSS-in-JS or styled-components | Conflicts with Tailwind CSS 4 build pipeline |
| Custom theme builder | Over-engineering; dark + light + system is sufficient |
| Icon font (Font Awesome) | 100-300KB for 15 icons; cannot tree-shake; FOIT/FOUT issues |
| Dynamic OG images per secret | Could leak secret metadata; security violation |
| Google Fonts CDN | Privacy violation for zero-knowledge app; self-host only |
| Analytics/tracking scripts | No third-party scripts; XSS risk and privacy violation |
