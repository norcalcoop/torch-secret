  <files>

  <action>
    First install the dependency:
    ```
    npm install posthog-js
    ```
    (Run from the repo root — posthog-js goes in the root package.json as a runtime dependency)

    Create `client/src/vite-env.d.ts` with Vite ImportMetaEnv augmentation:
    ```typescript
    /// <reference types="vite/client" />

    interface ImportMetaEnv {
      readonly VITE_POSTHOG_KEY?: string;
      readonly VITE_POSTHOG_HOST?: string;
    }

    interface ImportMeta {
      readonly env: ImportMetaEnv;
    }
    ```

    Create `client/src/analytics/posthog.ts` with the following structure:

    1. Import: `import posthog, { type CaptureResult } from 'posthog-js';`

    2. Module-level initialized flag:
    ```typescript
    let _initialized = false;
    function isInitialized(): boolean { return _initialized; }
    ```

    3. `stripFragment(value: unknown): string | null` — strips `#...` from URL strings. Uses `new URL()` try/catch for absolute URLs (sets `url.hash = ''` then returns `url.toString()`), falls back to `.replace(/#.*$/, '')` for relative URLs or non-URL strings. Returns null if value is not a string.

    4. `sanitizeEventUrls(event: CaptureResult | null): CaptureResult | null` — the `before_send` hook. Strips fragments from `$current_url`, `$referrer`, and `$initial_referrer` in `event.properties` if those keys are present. Returns the modified event. Returns null (dropping event) only if input is null.

    5. `initAnalytics(): void` — reads `import.meta.env.VITE_POSTHOG_KEY` and `import.meta.env.VITE_POSTHOG_HOST ?? 'https://us.i.posthog.com'`. Returns early (no-op) if key is falsy. Calls `posthog.init(key, { api_host: host, autocapture: false, disable_session_recording: true, capture_pageview: false, capture_pageleave: false, before_send: sanitizeEventUrls })`. Sets `_initialized = true` after init.

    6. `capturePageview(): void` — if not initialized, return. Calls `posthog.capture('$pageview', { $current_url: window.location.href })`.

    7. `captureSecretCreated(expiresIn: string, hasPassword: boolean): void` — if not initialized, return. Calls `posthog.capture('secret_created', { expires_in: expiresIn, has_password: hasPassword })`.
    ZERO-KNOWLEDGE INVARIANT: no userId, no secretId. Only non-identifying metadata.

    8. `captureSecretViewed(): void` — if not initialized, return. Calls `posthog.capture('secret_viewed')`.
    ZERO-KNOWLEDGE INVARIANT: no userId, no secretId.

    9. `captureUserRegistered(method: 'email' | 'google' | 'github'): void` — if not initialized, return. Calls `posthog.capture('user_registered', { method })`.
    ZERO-KNOWLEDGE INVARIANT: no secretId in this event.

    10. `captureUserLoggedIn(method: 'email' | 'google' | 'github'): void` — if not initialized, return. Calls `posthog.capture('user_logged_in', { method })`.

    11. `identifyUser(userId: string): void` — if not initialized, return. Calls `posthog.identify(userId)`.
    ZERO-KNOWLEDGE INVARIANT: pass userId only — never email, name, secretId, or any other PII.

    12. `resetAnalyticsIdentity(): void` — if not initialized, return. Calls `posthog.reset()`.

    Add a JSDoc block comment at the top of the file (before imports) explaining the zero-knowledge invariant: secret events must never contain userId; user events must never contain secretId.

    Export all public functions as named exports (no default export).
  </action>
