  <action>
    **app.ts:**
    Add import at top of file: `import { initAnalytics } from './analytics/posthog.js';`

    In the `DOMContentLoaded` callback, call `initAnalytics()` BEFORE `initThemeListener()`. This ensures PostHog is initialized before the router fires any route events. The updated init order is:
    ```typescript
    document.addEventListener('DOMContentLoaded', () => {
      initAnalytics();
      initThemeListener();
      createLayoutShell();
      initRouter();
    });
    ```

    **router.ts:**
    Add import at top of file (after existing imports): `import { capturePageview } from './analytics/posthog.js';`

    In `handleRoute()`, call `capturePageview()` immediately after the call to `updatePageMeta(...)` inside EACH route branch. Because every branch calls `updatePageMeta(...)` and then chains `.then(() => mod.renderXxxPage(container))`, the cleanest approach is to add a single `capturePageview()` call directly after the `window.dispatchEvent(new CustomEvent(...))` at the bottom of `handleRoute()`, which fires unconditionally on every route.

    Specifically: add `capturePageview();` immediately BEFORE the `window.dispatchEvent(new CustomEvent('routechange', { detail: { path } }));` line at the bottom of `handleRoute()`. This position ensures:
    1. `updatePageMeta()` has already been called (URL is correct in document metadata)
    2. The page module import has been dispatched (async, but URL is settled)
    3. On the reveal page, `history.replaceState` happens inside `renderRevealPage()` which is async — so `window.location.href` still contains the fragment at this point. This is acceptable because `before_send` in the analytics module strips the fragment from `$current_url` on every event. Belt-and-suspenders defense works here.

    The pageview call location should look like:
    ```typescript
    // Capture SPA pageview (before_send in analytics module strips any URL fragment)
    capturePageview();
    window.dispatchEvent(new CustomEvent('routechange', { detail: { path } }));
    ```
