Step 2 — Inspect the generated SQL file(s):

- `ALTER TABLE "secrets" ADD CONSTRAINT.*FOREIGN KEY ("user_id")`

Step 3 — Apply the bug #4147 workaround. There are two cases:

CASE A: drizzle-kit generated TWO separate files (one for auth tables, one for secrets FK):
- Rename the auth-tables file to `drizzle/0001_add_auth_tables.sql`
- Rename the secrets-FK file to `drizzle/0002_add_secrets_user_id.sql`
- Update _journal.json entries to use these exact tag names

CASE B: drizzle-kit generated ONE file containing both auth table creation AND secrets ALTER TABLE:
- Split the single generated file into two files manually:

  File `drizzle/0001_add_auth_tables.sql` — contains ONLY:
  - All `CREATE TABLE` statements for users, sessions, accounts, verification
  - All `ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY` statements linking sessions→users and accounts→users
  - All `CREATE INDEX` statements for accounts_user_id_idx and verification_identifier_idx
  - The drizzle marker comment at the top: `-- Current sql file was generated after introspecting the database`

  File `drizzle/0002_add_secrets_user_id.sql` — contains ONLY:
  - `ALTER TABLE "secrets" ADD COLUMN "user_id" text;` (ADD COLUMN must come before ADD CONSTRAINT)
  - `ALTER TABLE "secrets" ADD CONSTRAINT "secrets_user_id_users_id_fk" FOREIGN KEY ("user_id") REFERENCES "users"("id") ON DELETE set null ON UPDATE no action;`
  - `CREATE INDEX "secrets_user_id_created_at_idx" ON "secrets" ("user_id","created_at" DESC) WHERE "user_id" IS NOT NULL;`

  CRITICAL ORDER: In 0002, the `ADD COLUMN` statement MUST appear before `ADD CONSTRAINT`. This is the exact bug #4147 fix. Verify this before proceeding.

  Delete the original combined generated file.
