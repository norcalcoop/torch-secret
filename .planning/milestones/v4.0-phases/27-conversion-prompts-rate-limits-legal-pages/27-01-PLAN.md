**`createAnonHourlyLimiter(redisClient?: Redis)`**

- windowMs: 24 * 60 * 60 * 1000
- limit: isE2E ? 1000 : 10
- standardHeaders: false  ← CRITICAL: prevents daily limiter from overwriting hourly RateLimit-* headers; hourly limiter headers are what the client uses for prompt triggering
- legacyHeaders: false
- statusCode: 429
- message: `{ error: 'rate_limited', message: 'Daily limit reached for anonymous sharing. Create a free account for higher limits.' }`
- store: createStore(redisClient, 'rl:anon:d:')
- passOnStoreError: true
- skip: `(_req, res) => !!(res.locals.user as unknown)`

**`createAuthedDailyLimiter(redisClient?: Redis)`**
- windowMs: 24 * 60 * 60 * 1000
- limit: isE2E ? 1000 : 20
- standardHeaders: 'draft-7', legacyHeaders: false
- statusCode: 429
- message: `{ error: 'rate_limited', message: 'Daily limit reached. You can create 20 secrets per day.' }`
- store: createStore(redisClient, 'rl:authed:d:')
- passOnStoreError: true
- skip: `(_req, res) => !(res.locals.user as unknown)`  ← skips anonymous users
- keyGenerator: `(req, res) => (res.locals.user as { id: string }).id`  ← per-user counter, not per-IP (avoids shared-IP false positives for authenticated users)

Update the JSDoc comments on each factory to describe the limit, who it applies to, and the skip logic.

**In `server/src/routes/secrets.ts`:**

1. Update import from rate-limit.ts: replace `createSecretLimiter` with `createAnonHourlyLimiter, createAnonDailyLimiter, createAuthedDailyLimiter`.

2. Reorder the POST `/` route middleware chain from:
   ```
   createSecretLimiter(redisClient), optionalAuth, validateBody(CreateSecretSchema)
   ```
   to:
   ```
   optionalAuth,
   createAnonHourlyLimiter(redisClient),
   createAnonDailyLimiter(redisClient),
   createAuthedDailyLimiter(redisClient),
   validateBody(CreateSecretSchema),
   ```
   This ordering is CRITICAL — optionalAuth must run first so `res.locals.user` is set before any limiter's `skip` callback fires.

3. In the POST `/` async handler, add expiresIn tier enforcement immediately after reading `userId`:
   ```typescript
   // Enforce expiresIn caps per tier (server-side guard — Zod enum allows all values)
   // Anonymous users: max '1h'. Authenticated free users: max '7d'.
   const expiresIn = body.expiresIn;
   if (!userId && expiresIn !== '1h') {
     res.status(400).json({
       error: 'validation_error',
       message: 'Anonymous sharing is limited to 1-hour expiration. Sign up free for more options.',
     });
     return;
   }
   if (userId && expiresIn === '30d') {
     res.status(400).json({
       error: 'validation_error',
       message: 'Extended expiration (30 days) is a Pro feature. Free accounts can set up to 7 days.',
     });
     return;
   }
   ```

**In `client/src/api/client.ts`:**

1. Extend `ApiError` to carry an optional `rateLimitReset` field (Unix timestamp in seconds):
   ```typescript
   export class ApiError extends Error {
     readonly status: number;
     readonly body: unknown;
     readonly rateLimitReset?: number; // Unix timestamp (seconds) from RateLimit-Reset header

     constructor(status: number, body: unknown, rateLimitReset?: number) {
       super(`API error ${status}`);
       this.name = 'ApiError';
       this.status = status;
       this.body = body;
       this.rateLimitReset = rateLimitReset;
     }
   }
   ```

2. Update `createSecret()` to extract `RateLimit-Reset` header from 429 responses and attach it to the thrown `ApiError`:
   ```typescript
   export async function createSecret(...): Promise<CreateSecretResponse> {
     const res = await fetch('/api/secrets', { ... });

     if (!res.ok) {
       const body = await res.json() as unknown;
       if (res.status === 429) {
         const resetHeader = res.headers.get('RateLimit-Reset');
         const rateLimitReset = resetHeader ? parseInt(resetHeader, 10) : undefined;
         throw new ApiError(res.status, body, rateLimitReset);
       }
       throw new ApiError(res.status, body);
     }

     return res.json() as Promise<CreateSecretResponse>;
   }
   ```
   Keep all other API functions (getSecret, getSecretMeta, verifySecretPassword, fetchDashboardSecrets, deleteDashboardSecret) unchanged.
