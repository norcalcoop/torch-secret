<task type="auto">
  <name>Task 2: Clean up legacy tokens from styles.css</name>
  <files>client/src/styles.css</files>
  <action>
Now that all component files use semantic tokens (Task 1 complete), remove the legacy color tokens from styles.css:

1. Add `--color-*: initial;` as the FIRST line inside the `@theme inline` block. This clears Tailwind's default color palette (slate, gray, red, etc.) and the old primary-*/danger-500/etc tokens, preventing accidental use of non-semantic colors.

2. Remove any remaining legacy token declarations:
   - Remove `--color-primary-50` through `--color-primary-700`
   - Remove `--color-danger-500`, `--color-success-500`, `--color-warning-500`
   - These are now replaced by `--color-accent`, `--color-danger`, `--color-success`, `--color-warning`

3. Verify the final styles.css has a clean structure:
   - `@import "tailwindcss";`
   - `@import "@fontsource-variable/jetbrains-mono";`
   - `:root { color-scheme: dark; --ds-color-*: oklch(...); }` (all token values)
   - `@theme inline { --color-*: initial; --color-bg: var(--ds-color-bg); ... }` (semantic mappings only)
   - `@theme { --font-*: ...; --text-*: ...; }` (typography and scale)

4. Run vite build to confirm nothing broke from removing legacy tokens.
  </action>
