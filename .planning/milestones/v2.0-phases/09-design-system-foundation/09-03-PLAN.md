Create `client/src/components/icons.ts` with the following:

1. **Imports:**
   ```typescript
   import { createElement, type IconNode } from 'lucide';
   ```
   Import ONLY `createElement` and the `IconNode` type. Do NOT import any specific icon (like `Shield`, `Lock`) -- consumers import icons themselves and pass IconNode to createIcon.

2. **Size constants:**
   ```typescript
   export const ICON_SIZES = { sm: 16, md: 24, lg: 32 } as const;
   export type IconSize = keyof typeof ICON_SIZES;
   ```

3. **Options interface:**
   ```typescript
   export interface CreateIconOptions {
     /** Named size or pixel number. Default: 'md' (24px) */
     size?: IconSize | number;
     /** SVG stroke width. Default: 2 */
     strokeWidth?: number;
     /** CSS class(es) to add to the SVG element */
     class?: string | string[];
     /** If provided, sets aria-label and role="img" instead of aria-hidden */
     ariaLabel?: string;
   }
   ```

4. **createIcon function:**
   ```typescript
   export function createIcon(icon: IconNode, options: CreateIconOptions = {}): SVGSVGElement {
   ```
   Implementation:
   - Destructure options with defaults: `size = 'md'`, `strokeWidth = 2`
   - Resolve pixel size: `typeof size === 'number' ? size : ICON_SIZES[size]`
   - Build attrs object: `{ width: pixels, height: pixels, 'stroke-width': strokeWidth }`
   - Add default CSS class `'text-icon'` (uses the --color-icon token from styles.css). If `options.class` is provided, prepend 'text-icon' and append the user's classes.
   - If `ariaLabel` provided: set `attrs['aria-label'] = ariaLabel` and `attrs.role = 'img'`
   - If `ariaLabel` NOT provided: set `attrs['aria-hidden'] = 'true'` (decorative icon)
   - Call `createElement(icon, attrs)` and cast result to `SVGSVGElement`
   - Return the SVG element

5. **Re-export IconNode type** for consumer convenience:
   ```typescript
   export type { IconNode } from 'lucide';
   ```

6. **JSDoc comments** on all exports following codebase conventions (see CONVENTIONS.md).

**Critical implementation notes:**
- The `createElement` function from `lucide` takes `(iconNode, attrs)` and returns an SVG element. The attrs are plain objects with string/number values.
- The `class` attribute in Lucide's createElement uses the key `class` (not `className` -- it's SVG, not React).
- When passing class as array, Lucide joins them. When passing as string, use as-is.
- The `text-icon` class maps to `color: var(--color-icon)` via Tailwind's text-* utility, giving icons the dedicated icon color token.
- Do NOT use barrel imports from lucide (no `import { Shield, Lock, ... } from 'lucide'`). Consumers import individual icon data and pass to createIcon.

**Usage example (for JSDoc, not implemented here):**
```typescript
import { Shield } from 'lucide';
import { createIcon } from '../components/icons.js';

const icon = createIcon(Shield, { size: 'lg', ariaLabel: 'Security' });
container.appendChild(icon);
```

**Test cases (use `Shield` icon as test fixture):**

1. **"creates SVG element with default attributes"**
   - Call `createIcon(Shield)` with no options
   - Assert result is an SVGSVGElement (or at minimum has tagName 'svg')
   - Assert `getAttribute('width')` === '24' (md default)
   - Assert `getAttribute('height')` === '24'
   - Assert `getAttribute('stroke-width')` === '2'
   - Assert `getAttribute('aria-hidden')` === 'true'

2. **"applies named size variants"**
   - For each key in ICON_SIZES (sm, md, lg):
     - Call `createIcon(Shield, { size: key })`
     - Assert width/height match ICON_SIZES[key]

3. **"accepts custom numeric size"**
   - Call `createIcon(Shield, { size: 48 })`
   - Assert width === '48', height === '48'

4. **"sets aria-label and role when ariaLabel provided"**
   - Call `createIcon(Shield, { ariaLabel: 'Security shield' })`
   - Assert `getAttribute('aria-label')` === 'Security shield'
   - Assert `getAttribute('role')` === 'img'
   - Assert `getAttribute('aria-hidden')` is null (NOT set when label present)

5. **"applies custom CSS classes alongside default text-icon"**
   - Call `createIcon(Shield, { class: 'my-custom-class' })`
   - Assert the SVG's class attribute contains both 'text-icon' and 'my-custom-class'

6. **"applies array of CSS classes"**
   - Call `createIcon(Shield, { class: ['class-a', 'class-b'] })`
   - Assert class attribute contains 'text-icon', 'class-a', and 'class-b'

7. **"respects custom strokeWidth"**
   - Call `createIcon(Shield, { strokeWidth: 1.5 })`
   - Assert `getAttribute('stroke-width')` === '1.5'

**Note:** Lucide's createElement in happy-dom may create elements differently than in a real browser. If createElement returns a generic Element rather than SVGSVGElement, adjust assertions to check tagName === 'svg' rather than instanceof SVGSVGElement.

- Keep tests focused and independent
