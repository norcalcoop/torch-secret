<task type="auto">
  <name>Task 1: Install packages and create token system in styles.css</name>
  <files>package.json, package-lock.json, client/src/styles.css</files>
  <action>
1. Install the two new npm packages:
   ```
   npm install lucide @fontsource-variable/jetbrains-mono
   ```

2. Completely rewrite `client/src/styles.css` with this structure (in order):
   a. `@import "tailwindcss";`
   b. `@import "@fontsource-variable/jetbrains-mono";`
   c. `:root` block with `color-scheme: dark;` and ALL semantic design tokens as CSS custom properties using `--ds-` prefix and OKLCH values
   d. `@theme inline` block mapping `--color-*` Tailwind tokens to `var(--ds-color-*)` CSS variables
   e. `@theme` block (non-inline) for font stacks, type scale, and static values

**Token definitions (OKLCH values -- all pass WCAG AA against warm charcoal bg):**

`:root` block -- use the research-provided palette as starting point:
- Background: `--ds-color-bg` warm charcoal ~oklch(0.23 0.038 283)
- Surfaces: 3 levels (surface, surface-raised, surface-overlay) progressively lighter
- Text: 4 levels (primary ~0.96, secondary ~0.84, tertiary ~0.67, muted ~0.65) -- all must pass WCAG AA (4.5:1) against bg
- Accent: electric blue (oklch ~0.71 0.143 255) + hover state (darker, more saturated)
- Border: ~oklch(0.36 0.058 283)
- Status: danger (red), success (green), warning (amber) -- pick values that work with blue accent on warm charcoal
- Icon: dedicated `--ds-color-icon` token (~tertiary brightness)

`@theme inline` block:
- Reset defaults: `--color-*: initial;`
- Map each `--color-{name}` to `var(--ds-color-{name})`
- Keep `--color-white: #fff;` and `--color-black: #000;` for utilities that need them

`@theme` block (static values):
- `--font-heading`: `'JetBrains Mono Variable', ui-monospace, monospace`
- `--font-body`: `ui-sans-serif, system-ui, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji'`
- `--font-mono`: `'JetBrains Mono Variable', ui-monospace, 'Cascadia Code', 'Source Code Pro', monospace`
- Type scale (1.25 Major Third ratio from 1rem base): `--text-xs: 0.64rem`, `--text-sm: 0.8rem`, `--text-base: 1rem`, `--text-lg: 1.25rem`, `--text-xl: 1.563rem`, `--text-2xl: 1.953rem`, `--text-3xl: 2.441rem`

**Critical: Use `@theme inline` (not plain `@theme`) for all CSS variable references.** Plain `@theme` cannot reference `var()` -- this is a Tailwind v4 requirement. Use plain `@theme` only for static values like font stacks and type scale numbers.

**Critical: Use exact font family name `'JetBrains Mono Variable'`** (not `'JetBrains Mono'`) -- the variable font package registers under this name.

**Do NOT remove or rename** the existing `primary-50` through `primary-700`, `danger-500`, `success-500`, `warning-500` tokens yet. They may be referenced by existing code. Plan 02 handles the migration. Add new semantic tokens alongside old ones in this task.

Wait -- actually, the `@theme inline` block with `--color-*: initial` will clear all existing `--color-primary-*` tokens. This means we MUST either: (a) keep them by re-declaring in `@theme inline`, or (b) migrate all usages first. Since Plan 02 handles migration, we need to re-declare the old tokens temporarily so the app does not break.

**Re-declare legacy tokens in `@theme inline`** for backward compatibility:
```
--color-primary-50: var(--ds-color-accent-subtle);  /* or a light accent equivalent */
--color-primary-100: ...
```
Actually, the simplest approach: Do NOT use `--color-*: initial` yet. Instead, add the new semantic tokens alongside the existing ones. The `@theme inline` block should define ONLY the new semantic tokens (bg, surface, text-primary, accent, etc.) without clearing the namespace. Plan 02 will migrate all usages, and then a final cleanup in Plan 02 will add `--color-*: initial` and remove the legacy tokens.

**Revised approach for `@theme inline`:**
- Define new semantic tokens: `--color-bg`, `--color-surface`, `--color-surface-raised`, `--color-surface-overlay`, `--color-text-primary`, `--color-text-secondary`, `--color-text-tertiary`, `--color-text-muted`, `--color-accent`, `--color-accent-hover`, `--color-border`, `--color-danger`, `--color-success`, `--color-warning`, `--color-icon`
- Keep existing `@theme` block's `--color-primary-*`, `--color-danger-500`, `--color-success-500`, `--color-warning-500` tokens temporarily (move them into the `@theme inline` block since we need inline for variable references)
- Do NOT use `--color-*: initial` until Plan 02 completes migration

This way the app continues working with old classes while new semantic classes become available.
