<task type="auto">
  <name>Task 2: Add FOWT prevention script and create theme manager + toggle component</name>
  <files>client/index.html, client/src/theme.ts, client/src/components/theme-toggle.ts</files>
  <action>
**A. FOWT prevention script in `client/index.html`:**

Add a synchronous inline `<script>` as the FIRST element inside `<head>`, before the `<meta charset>` tag. This script MUST execute before ANY CSS loads to prevent flash of wrong theme:

```html
<script>
  (function() {
    var t = localStorage.getItem('theme');
    var d = t === 'dark' || (!t && matchMedia('(prefers-color-scheme:dark)').matches);
    document.documentElement.classList.toggle('dark', d);
    document.documentElement.style.colorScheme = d ? 'dark' : 'light';
  })();
</script>
```

Place it BEFORE `<meta charset="UTF-8" />`. The script reads localStorage synchronously, checks system preference, and applies the `.dark` class + `color-scheme` property on `<html>` before the browser paints.

**B. Theme manager module at `client/src/theme.ts`:**

Create a new file with these exports:

```typescript
export type ThemePreference = 'light' | 'dark' | 'system';
```

- `getThemePreference(): ThemePreference` â€” reads `localStorage.getItem('theme')`. Returns 'light' or 'dark' if stored, otherwise 'system'.

- `setThemePreference(pref: ThemePreference): void` â€” if 'system', removes the localStorage key; otherwise stores 'light' or 'dark'. Then calls `applyTheme()`.

- `applyTheme(): void` â€” reads preference, checks `matchMedia('(prefers-color-scheme: dark)').matches` for system mode, toggles `.dark` class on `document.documentElement`, and sets `document.documentElement.style.colorScheme` to 'dark' or 'light'.

- `initThemeListener(): void` â€” registers a `change` event listener on `matchMedia('(prefers-color-scheme: dark)')` that calls `applyTheme()` only when the current preference is 'system'. This ensures live OS theme changes are reflected.

Keep the module simple: ~40 lines, no classes, pure functions with module-level constants.

**C. Theme toggle component at `client/src/components/theme-toggle.ts`:**

Create a button component that cycles through light -> dark -> system on each click:

- Import `Sun`, `Moon`, `Monitor` from 'lucide' and `createIcon` from './icons.js'

- Define cycle order: `['light', 'dark', 'system']`
- Define icon map: `{ light: Sun, dark: Moon, system: Monitor }`
- Define label map: `{ light: 'Switch to dark mode', dark: 'Switch to system theme', system: 'Switch to light mode' }`
  (Labels describe what the NEXT click will do, per UX convention)

`createThemeToggle(): HTMLButtonElement`:
- Creates a `<button type="button">` with styling:
  `p-2 rounded-lg text-text-secondary hover:text-text-primary hover:bg-surface-raised transition-colors cursor-pointer focus:ring-2 focus:ring-accent focus:outline-hidden`
- Internal `render()` function: reads current preference, clears button children, appends the matching icon via `createIcon(ICONS[pref], { size: 'sm', class: 'text-text-secondary' })`, sets `aria-label` to the appropriate label.
- Click handler: gets current preference, computes next in cycle, calls `setThemePreference(next)`, calls `render()`.
- Calls `render()` once before returning.
- Returns the button element.

Export `createThemeToggle` as a named export.
