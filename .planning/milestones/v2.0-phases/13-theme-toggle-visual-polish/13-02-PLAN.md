**A. Page-enter animation in router (`client/src/router.ts`):**

In the `handleRoute()` function, after clearing the container and before the route-matching `if/else` block, add the `motion-safe:animate-fade-in-up` class to the `#app` container so every page render gets the fade-in-up animation:

```typescript
// Clear existing content
while (container.firstChild) {
  container.removeChild(container.firstChild);
}

// Add page-enter animation (respects prefers-reduced-motion via motion-safe:)
container.classList.remove('motion-safe:animate-fade-in-up');
// Force reflow to restart animation on same-route navigations
void container.offsetWidth;
container.classList.add('motion-safe:animate-fade-in-up');
```

The `motion-safe:` prefix ensures users who prefer reduced motion see no animation. The remove-reflow-add pattern ensures the animation plays on every navigation, not just the first.

**B. Copy-button icon swap animation (`client/src/components/copy-button.ts`):**

Transform the copy button to use Lucide icons (Copy/Check) instead of text-only, with an icon swap animation on success:

2. Refactor `createCopyButton` to include an icon:
   - After creating the button, add the Copy icon and label text as children instead of just `textContent`:
     ```typescript
     const iconSpan = document.createElement('span');

     button.textContent = ''; // Clear the textContent that was set earlier
     ```
     Actually, restructure: remove the `button.textContent = defaultLabel;` line entirely. Instead build the button content as icon + label from the start.

3. Add icon swap helper function (module-level):
   ```typescript
   function swapToCheckIcon(iconSpan: HTMLElement): void {
     // Respect reduced motion preference
     const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

     // Replace icon
     while (iconSpan.firstChild) iconSpan.removeChild(iconSpan.firstChild);
     iconSpan.appendChild(createIcon(Check, { size: 'sm', class: 'text-white' }));

     // Add subtle scale animation if motion is allowed
     if (!prefersReduced) {
       iconSpan.classList.add('transition-transform', 'duration-150');
       iconSpan.classList.add('scale-110');
       setTimeout(() => iconSpan.classList.remove('scale-110'), 150);
     }

     // Revert to Copy icon after 1.5 seconds
     setTimeout(() => {
       while (iconSpan.firstChild) iconSpan.removeChild(iconSpan.firstChild);
       iconSpan.appendChild(createIcon(Copy, { size: 'sm', class: 'text-white' }));
     }, 1500);
   }
   ```

4. In the click handler, after successful copy (both clipboard API and fallback paths), call `swapToCheckIcon(iconSpan)` in addition to `showToast('Copied to clipboard')`.

5. Keep the `iconSpan` reference accessible in the click handler via closure (it's created in the same `createCopyButton` function scope).

**IMPORTANT:** The icon color class should match the button context. For the primary copy button, use `'text-white'`. The terminal block's copy button has its own overridden styling, and since it also calls `createCopyButton`, the icon swap will work there too. However, the terminal block overrides `copyBtn.className` after creation, so the icon inside will inherit the button's text color naturally.

Also gate the dismiss transition: in the `setTimeout` callback that adds opacity transition classes, check reduced motion:
```typescript
setTimeout(() => {
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  if (prefersReduced) {
    // Instant removal for reduced motion users
    toast.remove();
  } else {
    toast.classList.add('opacity-0', 'transition-opacity', 'duration-200');
    toast.addEventListener('transitionend', () => { toast.remove(); }, { once: true });
  }
}, durationMs);
```
