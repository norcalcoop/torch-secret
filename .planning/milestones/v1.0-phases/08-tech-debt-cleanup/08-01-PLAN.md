<objective>
Fix flaky test failures and test parallelism issues by migrating vitest config from flat `test` block to `projects` array. Server tests run sequentially (fileParallelism: false) to eliminate the shared pool race condition; client tests keep parallel execution.

Purpose: Items 1 and 3 from the v1 milestone audit share a single root cause -- 4 server test files import the same `pool` singleton from `connection.ts`, and each calls `pool.end()` in `afterAll`. When Vitest runs files in parallel, one file's teardown closes the shared pool while another file's tests are still executing.

Output: Updated `vitest.config.ts` with projects-based configuration. All tests pass reliably.
</objective>

  <action>
Replace the flat `test` configuration in `vitest.config.ts` with a `projects` array containing two project entries:

1. **client** project:
   - `name: 'client'`
   - `include: ['client/src/**/*.test.ts']`
   - `environment: 'happy-dom'` (replaces the `environmentMatchGlobs` approach which is unreliable in vitest 4.x for nested paths, per decision [07-02])
   - No `fileParallelism` override (keeps default parallel execution)

2. **server** project:
   - `name: 'server'`
   - `include: ['server/src/**/*.test.ts']`
   - `environment: 'node'`
   - `fileParallelism: false` (runs test files sequentially to prevent the shared pool race condition)

Keep the following at the top-level `test` block (shared across both projects):
   - `globals: false`
   - `testTimeout: 10_000`
   - `setupFiles: ['dotenv/config']`

Remove the `include` and `environmentMatchGlobs` fields from the top-level `test` block since each project now specifies its own.

The resulting config should look like this:

```typescript
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: false,
    testTimeout: 10_000,
    setupFiles: ['dotenv/config'],
    projects: [
      {
        test: {
          name: 'client',
          include: ['client/src/**/*.test.ts'],
          environment: 'happy-dom',
        },
      },
      {
        test: {
          name: 'server',
          include: ['server/src/**/*.test.ts'],
          environment: 'node',
          fileParallelism: false,
        },
      },
    ],
  },
});
```

Do NOT remove `pool.end()` calls from individual test files -- they are harmless when files run sequentially and serve as defensive cleanup.

Do NOT remove `@vitest-environment happy-dom` comment directives from client test files -- they are redundant with the projects config but harmless, and removing them is out of scope.
