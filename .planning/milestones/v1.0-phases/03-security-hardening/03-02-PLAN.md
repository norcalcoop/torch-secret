**Describe: "Success Criterion 1: Content-Security-Policy with nonce"**
- Test: "CSP header includes nonce-based script-src" -- POST /api/secrets, assert `content-security-policy` header exists, contains `script-src 'self' 'nonce-`, does NOT contain `unsafe-inline`, does NOT contain `unsafe-eval`.
- Test: "CSP nonce is unique per request" -- make two POST requests, extract nonce from each CSP header using regex `/nonce-([a-f0-9]+)/`, assert nonces are different.
- Test: "CSP header includes style-src with nonce" -- assert CSP contains `style-src 'self' 'nonce-`.
- Test: "CSP blocks object and frame embedding" -- assert CSP contains `object-src 'none'` and `frame-ancestors 'none'`.

**Describe: "Success Criterion 2: Rate limiting on POST /api/secrets"**
- Test: "returns 429 after 10 POST requests in same window" -- create a FRESH app instance via `buildApp()` inside this test (so rate limit counter starts at 0). Send 10 POST requests, all should return 201. Send 11th, expect 429 with body `{ error: 'rate_limited' }`.
- Test: "GET requests are not rate-limited" -- use a separate app instance. Send 15 GET requests to `/api/secrets/xxxxxxxxxxxxxxxxxxx01` (invalid but not rate-limited). None should return 429 (they return 400 for bad ID format, which is fine -- just not 429).
- Test: "rate limit headers present on responses" -- POST to /api/secrets, assert `ratelimit-limit` header exists (draft-7 standard header).

**Describe: "Success Criterion 3: HTTPS redirect and HSTS"**
- Test: "HSTS header present in production mode" -- note: this test is tricky because the app reads NODE_ENV at middleware creation time. Instead, test that the `strict-transport-security` header is either present or correctly conditional. If NODE_ENV is 'test', HSTS may be disabled (which is correct behavior per research). Assert the app's behavior matches the environment.
  - Approach: If `process.env.NODE_ENV !== 'production'`, assert HSTS header is NOT present (correctly disabled in dev/test). Document that production would enable it.
- Test: "HTTPS redirect skipped in non-production" -- GET request should NOT redirect (no 301), confirming the conditional skip works for test/development environments.

**Describe: "Success Criterion 4: Referrer-Policy"**
- Test: "Referrer-Policy is no-referrer on POST response" -- POST /api/secrets, assert `referrer-policy` header is `'no-referrer'`.
- Test: "Referrer-Policy is no-referrer on GET response" -- GET /api/secrets/:id (create one first, then retrieve), assert `referrer-policy` header is `'no-referrer'`.

**Describe: "Success Criterion 5: Same-origin CORS"**
- Test: "no Access-Control-Allow-Origin header on same-origin request" -- POST /api/secrets without Origin header, assert `access-control-allow-origin` header is undefined.
- Test: "no Access-Control-Allow-Origin header on cross-origin request" -- POST /api/secrets with `Origin: https://evil.com`, assert `access-control-allow-origin` header is undefined.
- Test: "preflight OPTIONS from foreign origin gets no CORS headers" -- OPTIONS /api/secrets with `Origin: https://attacker.com` and `Access-Control-Request-Method: POST`, assert both `access-control-allow-origin` and `access-control-allow-methods` are undefined.

**Rate limit test isolation:** The rate limiter uses an in-memory store per app instance. Create a fresh `buildApp()` for the rate limit describe block to avoid interference from other tests. Other describe blocks can share a single app instance at the top of the file.
