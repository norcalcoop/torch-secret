Install helmet and express-rate-limit:
```bash
npm install helmet express-rate-limit
```

Create `server/src/middleware/security.ts` with three exports:

1. **`cspNonceMiddleware`** -- generates a per-request CSP nonce using `crypto.randomBytes(32)` (async callback version), stores as hex string in `res.locals.cspNonce`, then calls `next()`. On error, calls `next(err)`. Import `crypto` from `node:crypto`.

2. **`createHelmetMiddleware()`** -- returns `helmet()` configured with:
   - `contentSecurityPolicy.directives`: `defaultSrc: ["'self'"]`, `scriptSrc: ["'self'", (req, res) => \`'nonce-${res.locals.cspNonce}'\`]`, `styleSrc: ["'self'", (req, res) => \`'nonce-${res.locals.cspNonce}'\`]`, `imgSrc: ["'self'"]`, `connectSrc: ["'self'"]`, `fontSrc: ["'self'"]`, `objectSrc: ["'none'"]`, `frameAncestors: ["'none'"]`, `formAction: ["'self'"]`, `baseUri: ["'self'"]`, `upgradeInsecureRequests: []`
   - `strictTransportSecurity`: `maxAge: 31536000` (1 year), `includeSubDomains: true`, `preload: false`. Conditionally disable entirely when `NODE_ENV !== 'production'` by setting `strictTransportSecurity: false` to prevent browser HSTS lockout in development.
   - `referrerPolicy`: `{ policy: 'no-referrer' }` -- prevents URL fragment leakage
   - Explicitly enable: `xContentTypeOptions: true`, `crossOriginOpenerPolicy: { policy: 'same-origin' }`, `crossOriginResourcePolicy: { policy: 'same-origin' }`

3. **`httpsRedirect`** -- Express middleware that:
   - Skips (`next()`) when `process.env.NODE_ENV !== 'production'`
   - When `!req.secure`, responds with `301` redirect to `https://${req.hostname}${req.originalUrl}`
   - Otherwise calls `next()`

Import `env` from `../config/env.js` for the NODE_ENV check in helmet HSTS configuration. Use `process.env.NODE_ENV` directly in the httpsRedirect middleware (simpler, no circular dependency risk).

Type annotations: use `Request`, `Response`, `NextFunction` from `express`. Helmet's directive function signature is `(_req: Request, res: Response) => string`.

Note on CJS interop: helmet is ESM-compatible in v8.x. Use `import helmet from 'helmet';` directly (no CJS workaround needed unlike pino-http).

Create `server/src/middleware/rate-limit.ts` with one export:

**`createSecretLimiter`** -- `rateLimit()` configured with:
- `windowMs: 60 * 60 * 1000` (1 hour)
- `limit: 10` (10 requests per window per IP)
- `standardHeaders: 'draft-7'` (modern RateLimit headers)
- `legacyHeaders: false` (no X-RateLimit-* headers)
- `statusCode: 429`
- `message: { error: 'rate_limited', message: 'Too many secrets created. Please try again later.' }`
- Default `keyGenerator` (uses `req.ip`, which works correctly with trust proxy)

Import `{ rateLimit }` from `express-rate-limit` (named export).

Update `server/src/app.ts` to register security middleware in this exact order (order is critical for correctness):

```
1. app.set('trust proxy', 1)              -- FIRST: enables correct req.ip behind reverse proxy
2. app.use(httpsRedirect)                  -- redirect HTTP->HTTPS before any response
3. app.use(cspNonceMiddleware)             -- generate nonce BEFORE helmet reads it
4. app.use(createHelmetMiddleware())        -- set all security headers (CSP uses nonce from step 3)
5. app.use(httpLogger)                     -- existing: logging
6. app.use(express.json({ limit: '100kb' }))  -- existing: body parsing
7. app.use('/api/secrets', secretsRouter)  -- existing: routes
8. app.use(errorHandler)                   -- existing: MUST be last
```

Add imports at top of app.ts:
- `import { cspNonceMiddleware, createHelmetMiddleware, httpsRedirect } from './middleware/security.js';`

The rate limiter is route-scoped, NOT global. Update `server/src/routes/secrets.ts`:
- Import `{ createSecretLimiter }` from `'../middleware/rate-limit.js'`
- Add `createSecretLimiter` as middleware on the POST route only:
  `secretsRouter.post('/', createSecretLimiter, validateBody(CreateSecretSchema), async (req, res) => { ... })`
- Do NOT add rate limiter to GET route (retrieving a secret is a one-time operation, not abuse-prone)

Do NOT install or import the `cors` npm package. Express without cors middleware sends no CORS headers, which enforces same-origin by default. This is the correct behavior for SECR-06.
