<task type="auto">
  <name>Task 1: Install dependencies and configure server TypeScript</name>
  <files>
    package.json
    server/tsconfig.json
  </files>
  <action>
1. Add `"type": "module"` to the root package.json (required for nanoid 5.x ESM-only).

2. Install production dependencies:
   ```
   npm install express@5 pg drizzle-orm nanoid zod pino pino-http dotenv
   ```

3. Install dev dependencies:
   ```
   npm install -D drizzle-kit @types/express@5 @types/pg @types/supertest supertest tsx pino-pretty
   ```

4. Add scripts to package.json:
   ```json
   "scripts": {
     "dev:server": "tsx watch server/src/server.ts",
     "db:generate": "drizzle-kit generate",
     "db:migrate": "drizzle-kit migrate",
     "test": "vitest",
     "test:run": "vitest run"
   }
   ```

5. Create `server/tsconfig.json` extending root but configured for Node.js ESM:
   ```json
   {
     "extends": "../tsconfig.json",
     "compilerOptions": {
       "module": "NodeNext",
       "moduleResolution": "NodeNext",
       "outDir": "../dist/server",
       "rootDir": "src",
       "lib": ["ES2022"],
       "types": ["node"]
     },
     "include": ["src/**/*.ts"],
     "exclude": ["node_modules"]
   }
   ```
   Key: `module: "NodeNext"` for proper Node.js ESM (requires `.js` extensions in imports). Remove `"DOM"` from lib since server has no browser APIs. Use `"types": ["node"]` instead of `"vitest/globals"`.

6. Update root `tsconfig.json` to add `"server/src/**/*.ts"` to the include array (alongside existing `"client/src/**/*.ts"` and `"shared/**/*.ts"`).
  </action>
  <verify>
    Run `npx tsc --project server/tsconfig.json --noEmit` -- should produce no errors (once source files exist from Task 2).
    Run `node -e "import('nanoid').then(m => console.log(m.nanoid()))"` -- should print a 21-char ID (confirms ESM works).
  </verify>
  <done>All dependencies installed, server tsconfig compiles, ESM imports work, npm scripts defined.</done>
</task>

  <name>Task 2: Create database schema, connection, env config, logger, and shared types</name>

  <action>
1. **`.env.example`** -- Create with documented env vars (no real values):
   ```
   DATABASE_URL=postgresql://user:password@localhost:5432/secureshare
   PORT=3000
   LOG_LEVEL=info
   NODE_ENV=development
   ```

2. **`server/src/config/env.ts`** -- Zod-validated environment config:
   - Import `dotenv/config` at top (side-effect import to load .env)
   - Define Zod schema: `DATABASE_URL` (string, url starting with postgres), `PORT` (coerce to number, default 3000), `LOG_LEVEL` (enum: fatal/error/warn/info/debug/trace, default info), `NODE_ENV` (enum: development/production/test, default development)
   - Parse `process.env` through the schema
   - Export the validated `env` object
   - Export the `EnvSchema` type

3. **`server/src/db/schema.ts`** -- Drizzle secrets table:
   - Import `pgTable, text, timestamp, integer` from `drizzle-orm/pg-core`
   - Import `nanoid` from `nanoid`
   - Define `secrets` table with columns:
     - `id`: text primary key, `$defaultFn(() => nanoid())` -- 21-char nanoid
     - `ciphertext`: text, notNull -- base64-encoded blob from Phase 1
     - `expiresAt`: timestamp with timezone, notNull
     - `createdAt`: timestamp with timezone, defaultNow, notNull
     - `passwordHash`: text, nullable (for Phase 5)
     - `passwordAttempts`: integer, default 0, notNull (for Phase 5)
   - Export `secrets` table and infer select/insert types using `typeof secrets.$inferSelect` and `typeof secrets.$inferInsert`

4. **`server/src/db/connection.ts`** -- Database pool:
   - Import Pool from `pg`
   - Import `drizzle` from `drizzle-orm/node-postgres`
   - Import schema
   - Import `env` from config
   - Create `pool` with `connectionString: env.DATABASE_URL`
   - Create and export `db = drizzle({ client: pool, schema })`
   - Export `pool` for graceful shutdown

5. **`server/src/db/migrate.ts`** -- Programmatic migration:
   - Import drizzle and migrate from `drizzle-orm/node-postgres/migrator`
   - Connect using env.DATABASE_URL
   - Run `migrate(db)` which reads from `./drizzle` folder
   - Log success/failure and exit

6. **`server/src/middleware/logger.ts`** -- Pino logger with redaction:
   - Import pino and pinoHttp
   - Import `env` from config
   - Create base `logger` with level from env.LOG_LEVEL
   - Configure `redact.paths` for `req.headers.authorization`, `req.headers.cookie`, `res.headers["set-cookie"]`
   - Create `httpLogger` with custom request serializer that replaces `/api/secrets/ANYTHING` with `/api/secrets/[REDACTED]` in the URL field. Use regex: `/\/api\/secrets\/[A-Za-z0-9_-]+/g`
   - Do NOT log request bodies (could contain ciphertext) -- set `wrapSerializers: false` and only serialize method + redacted URL
   - Export both `logger` and `httpLogger`

7. **`shared/types/api.ts`** -- Shared Zod schemas and types:
   - Import `z` from `zod`
   - `CreateSecretSchema`: object with `ciphertext` (string, min 1, max 200000) and `expiresIn` (enum: '1h', '24h', '7d', '30d')
   - `SecretIdParamSchema`: object with `id` (string, length 21)
   - Export inferred types: `CreateSecretRequest`, `SecretIdParam`
   - Define and export response types (plain TS interfaces, not Zod):
     - `CreateSecretResponse`: `{ id: string; expiresAt: string }`
     - `SecretResponse`: `{ ciphertext: string; expiresAt: string }`
     - `ErrorResponse`: `{ error: string; message: string }`

8. **`drizzle.config.ts`** -- Drizzle Kit configuration:
   - Import `dotenv/config` and `defineConfig` from `drizzle-kit`
   - Configure: dialect 'postgresql', schema path `./server/src/db/schema.ts`, out `./drizzle`, dbCredentials with url from `process.env.DATABASE_URL!`
  </action>

    Verify logger redaction regex works: the pattern `/api/secrets/abc123` should become `/api/secrets/[REDACTED]` (will be tested in Plan 03 integration tests).
  </verify>
  <done>Database schema defines all 6 columns. Connection pool configured. Env validated with Zod. Logger redacts secret IDs. Shared API types defined with Zod schemas. Drizzle config ready for migrations. Server TypeScript compiles cleanly.</done>
