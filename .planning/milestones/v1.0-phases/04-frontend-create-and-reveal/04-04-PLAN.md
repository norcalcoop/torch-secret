  </what-built>
  <how-to-verify>
    Prerequisites: PostgreSQL running, `.env` configured with DATABASE_URL.

    1. Start the backend: `npm run dev:server` (Express on port 3000)
    2. Start the frontend: `npm run dev:client` (Vite on port 5173)
    3. Open http://localhost:5173 in your browser

    **Test 1: Create Flow**
    - You should see a textarea, expiration dropdown (defaulting to 24h), and "Create Secure Link" button
    - Type a test secret (e.g., "Hello, this is a secret message!")
    - Click "Create Secure Link"
    - You should see a confirmation page with a URL containing /secret/...#...
    - Click "Copy Link" and verify it copies to clipboard
    - Note the expiration timestamp shown

    **Test 2: Reveal Flow**
    - Open the copied link in a new tab (or incognito window)
    - You should see "You've received a secret" with a "Reveal Secret" button
    - Check the URL bar: the #key fragment should NOT be visible (stripped immediately)
    - Click "Reveal Secret"
    - You should see the decrypted secret text matching what you typed
    - Click "Copy Secret" to verify copy button works

    **Test 3: One-Time View**
    - Try opening the same link again in another tab
    - You should see an error message: "This secret is no longer available..."

    **Test 4: Mobile Responsive**
    - Open browser dev tools, toggle device toolbar
    - Set viewport to 375px width (iPhone SE)
    - Navigate to http://localhost:5173 and verify the create page is usable
    - Verify text is readable, buttons are tappable, no horizontal scroll

    **Test 5: Error States**
    - Visit http://localhost:5173/secret/invalidid123456789ab (wrong ID format or nonexistent)
    - Visit http://localhost:5173/secret/aaaaabbbbbcccccddddde (valid format, nonexistent -- should show generic error)
    - Visit http://localhost:5173/nonexistent-page (should show 404)
