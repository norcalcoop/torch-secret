      const res = await fetch(`/api/secrets/${id}/meta`);
      if (!res.ok) throw new ApiError(res.status, await res.json());
      return res.json() as Promise<MetaResponse>;
    }

    Add `verifySecretPassword`:
    ```typescript
    export async function verifySecretPassword(
      id: string,
      password: string,
    ): Promise<VerifySecretResponse> {
      const res = await fetch(`/api/secrets/${id}/verify`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ password }),
      });
      if (!res.ok) throw new ApiError(res.status, await res.json());
      return res.json() as Promise<VerifySecretResponse>;
    }
    ```

    Create a `renderPasswordEntry` function inside `renderRevealPage` (closure scope, same pattern as `renderInterstitial`):

    ```typescript

      encryptionKey: string,
      attemptsRemaining: number,
    ): void {
    ```

    The password entry form should include:
    - Lock icon (same style as interstitial icon)
    - Heading: "Password Required"
    - Subtext: "This secret is password protected. Enter the password to reveal it."
    - Attempt counter: `<p>` showing "{N} attempts remaining" (or "1 attempt remaining" for singular). Use `text-warning-500` or `text-danger-500` when N <= 1 for visual urgency. Give it an `id` or variable reference for dynamic updates.
    - Password input: `<input type="password">` with label, matching existing input styling (min-h-[44px], border, focus ring, etc.)
    - Submit button: "Verify Password" (same styling as "Reveal Secret" button)
    - Error message area: hidden div with `role="alert"`, shown on wrong password

    On form submit:
    1. Disable input + button, show loading state ("Verifying...")
    2. Call `verifySecretPassword(secretId, passwordValue)`
    3. On success: receive ciphertext, decrypt with `decrypt(ciphertext, encryptionKey)`, call `renderRevealedSecret(container, plaintext)`, set key = null
    4. On `ApiError` with status 403: parse `body.attemptsRemaining` from the error body. Update the attempt counter text. Show error "Wrong password. {N} attempts remaining." in the error area. Re-enable form.
    5. On `ApiError` with status 403 and `body.attemptsRemaining === 0`: `renderErrorPage(container, 'destroyed')` (secret was auto-destroyed)
    6. On `ApiError` with status 404: `renderErrorPage(container, 'not_available')` (secret gone -- maybe concurrent destroy)
    7. On any other error: `renderErrorPage(container, 'not_available')`

    CRITICAL SECURITY NOTES:
    - The encryption key (`key` variable) must remain in closure scope and be nulled after successful decryption, same as the existing handleReveal pattern.
    - Do NOT call `getSecret` for password-protected secrets -- the backend rejects it. Use `verifySecretPassword` exclusively.
    - The `renderRevealedSecret` function is already defined at module scope (outside `renderRevealPage`). It can be called from within the password flow.
