  <action>
    **1. Create `server/src/services/password.service.ts`:**

    Install argon2 first: `npm install argon2`

    Thin wrapper around the argon2 npm package (v0.44.0). Two exported functions:

    - `hashPassword(password: string): Promise<string>` -- calls `argon2.hash(password, { type: argon2.argon2id, memoryCost: 19456, timeCost: 2, parallelism: 1 })`. These are OWASP-recommended Argon2id parameters.
    - `verifyPassword(hash: string, password: string): Promise<boolean>` -- calls `argon2.verify(hash, password)`. The argon2 library uses constant-time comparison internally (PASS-05).

    Import argon2 with `import argon2 from 'argon2'` (default export, CJS interop like pg).

    **2. Update `shared/types/api.ts`:**

    Add optional password to CreateSecretSchema:
    ```typescript
    password: z.string().min(1).max(128).optional(),
    ```

    Add new schemas and interfaces:
    ```typescript
    export const VerifySecretSchema = z.object({
      password: z.string().min(1).max(128),
    });
    export type VerifySecretRequest = z.infer<typeof VerifySecretSchema>;

    export interface MetaResponse {
      requiresPassword: boolean;
      passwordAttemptsRemaining: number;
    }

    export interface VerifySecretResponse {
      ciphertext: string;
      expiresAt: string;
    }

    export interface VerifyErrorResponse {
      error: 'wrong_password';
      attemptsRemaining: number;
    }
    ```

    **3. Update `server/src/services/secrets.service.ts`:**

    Import `hashPassword` and `verifyPassword` from `./password.service.js`. Import `sql` from `drizzle-orm`.

    Modify `createSecret` signature to accept optional password:
    ```typescript
    export async function createSecret(
      ciphertext: string,
      expiresIn: '1h' | '24h' | '7d' | '30d',
      password?: string,
    ): Promise<Secret> {
    ```
    If password is provided, hash it with `hashPassword(password)` and include `passwordHash` in the `.values()` call. If not provided, leave passwordHash as null (schema default).

    Add `getSecretMeta` function:
    ```typescript
    export async function getSecretMeta(id: string): Promise<{
      requiresPassword: boolean;
      passwordAttemptsRemaining: number;
    } | null> {
    ```
    SELECT only `passwordHash` and `passwordAttempts` columns (plus `id` for the where clause). Returns null if secret doesn't exist. Returns `{ requiresPassword: passwordHash !== null, passwordAttemptsRemaining: Math.max(0, 3 - passwordAttempts) }`.

    Add `verifyAndRetrieve` function:
    ```typescript
    export async function verifyAndRetrieve(
      id: string,
      password: string,
    ): Promise<{ success: true; secret: Secret } | { success: false; attemptsRemaining: number } | null> {
    ```
    Uses `db.transaction`:
    1. SELECT the full secret row
    2. If not found, return null
    3. If no passwordHash (should not happen -- route guards prevent this), return null
    4. Call `verifyPassword(secret.passwordHash!, password)`
    5. If password WRONG:
       - Atomically increment passwordAttempts: `sql\`${secrets.passwordAttempts} + 1\``
       - Read the new attempt count. If `newAttempts >= 3`: zero ciphertext then delete (reuse the zero-then-delete pattern from retrieveAndDestroy), return `{ success: false, attemptsRemaining: 0 }`
       - Otherwise return `{ success: false, attemptsRemaining: 3 - newAttempts }`
    6. If password CORRECT: zero ciphertext, delete row (same pattern), return `{ success: true, secret }` (with original ciphertext from step 1)

    Modify `retrieveAndDestroy` to REJECT password-protected secrets: after the SELECT, if `secret.passwordHash !== null`, return null. This prevents bypassing password protection via the existing GET /:id endpoint.

    Add a `verifySecretLimiter` factory function, following the same pattern as `createSecretLimiter`:
    ```typescript
    export function verifySecretLimiter() {
      return rateLimit({
        windowMs: 15 * 60 * 1000,  // 15 minutes
        limit: 15,                  // 15 attempts per window per IP
        standardHeaders: 'draft-7',
        legacyHeaders: false,
        statusCode: 429,
        message: {
          error: 'rate_limited',
          message: 'Too many password attempts. Please try again later.',
        },
      });
    }
    ```

    **2. Update `server/src/routes/secrets.ts`:**

    Import new dependencies:
    - `getSecretMeta, verifyAndRetrieve` from secrets.service.js
    - `VerifySecretSchema` from shared/types/api.js
    - `verifySecretLimiter` from middleware/rate-limit.js

    Modify the existing POST handler to pass `req.body.password` to `createSecret`:
    ```typescript
    const secret = await createSecret(req.body.ciphertext, req.body.expiresIn, req.body.password);
    ```

    Add GET `/:id/meta` endpoint:
    - Validates params with `SecretIdParamSchema`
    - Calls `getSecretMeta(req.params.id)`
    - If null: returns 404 with `SECRET_NOT_AVAILABLE`
    - If found: returns 200 with `{ requiresPassword, passwordAttemptsRemaining }`

    Add POST `/:id/verify` endpoint:
    - Route middleware: `verifySecretLimiter()`, `validateParams(SecretIdParamSchema)`, `validateBody(VerifySecretSchema)`
    - Calls `verifyAndRetrieve(req.params.id, req.body.password)`
    - If null (not found): returns 404 with `SECRET_NOT_AVAILABLE`
    - If `{ success: false, attemptsRemaining: 0 }`: returns 404 with `SECRET_NOT_AVAILABLE` (secret destroyed, same response as nonexistent)
    - If `{ success: false, attemptsRemaining: N }` where N > 0: returns 403 with `{ error: 'wrong_password', attemptsRemaining: N }`
    - If `{ success: true, secret }`: returns 200 with `{ ciphertext: secret.ciphertext, expiresAt: secret.expiresAt.toISOString() }`

    IMPORTANT: The `/:id/meta` route must be registered BEFORE `/:id` to avoid Express matching `meta` as an `:id` parameter. Similarly, `/:id/verify` must be registered before the catch-all `/:id`. Order the routes: POST `/`, GET `/:id/meta`, POST `/:id/verify`, GET `/:id`.
