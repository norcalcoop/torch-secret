## Security Invariants

All zero-knowledge security invariants maintained across all phases:

- ✓ Server never sees plaintext secrets
- ✓ Encryption key lives only in URL fragment (never sent to server)
- ✓ Atomic read-and-destroy (SELECT → ZERO → DELETE in single transaction)
- ✓ Anti-enumeration (identical 404 responses across all error conditions)
- ✓ No PII in application logs (secret IDs redacted via regex)
- ✓ CSP nonce prevents XSS (no unsafe-inline, no unsafe-eval)
- ✓ Referrer-Policy prevents fragment leakage
