**File 1: `e2e/specs/password-flow.spec.ts`**

Import `test` and `expect` from `'../fixtures/test'`.

`test.describe('Password-protected secret flow', () => { ... })` containing:

**Test 1: "create with password, reveal with correct password"**
1. `const secret = await createTestSecret({ plaintext: 'Password protected E2E secret', password: 'mypassword123' })`
2. `await page.goto(secret.url)`
3. Wait for heading "Password Required": `await expect(page.getByRole('heading', { name: 'Password Required' })).toBeVisible()`
4. Fill the password field labeled "Password" with `'mypassword123'`: `await page.getByLabel('Password').fill('mypassword123')`
5. Click the "Verify Password" button: `await page.getByRole('button', { name: 'Verify Password' }).click()`
6. Wait for "Secret Revealed" heading: `await expect(page.getByRole('heading', { name: 'Secret Revealed' })).toBeVisible()`
7. Assert plaintext is visible: `await expect(page.getByText('Password protected E2E secret')).toBeVisible()`

**Test 2: "wrong password shows error with attempts remaining"**
1. `const secret = await createTestSecret({ plaintext: 'Another secret', password: 'correctpassword' })`
2. `await page.goto(secret.url)`
3. Wait for "Password Required" heading
4. Fill password with `'wrongpassword'`
5. Click "Verify Password"
6. Wait for error text: `await expect(page.getByText(/[Ww]rong password/)).toBeVisible()`
7. Assert attempts remaining visible: `await expect(page.getByText(/\d+ attempts? remaining/)).toBeVisible()`

NOTE: Each test creates a FRESH secret via the API fixture. Secrets are one-time-use, so they cannot be shared between tests.

Import `test` and `expect` from `'../fixtures/test'`.

`test.describe('Error states', () => { ... })` containing:

**Test 1: "already viewed secret shows error"**
1. `const secret = await createTestSecret()` -- creates a regular (no password) secret
2. First view -- consume the secret:
   - `await page.goto(secret.url)`
   - `await page.getByRole('button', { name: 'Reveal Secret' }).click()`
   - `await expect(page.getByRole('heading', { name: 'Secret Revealed' })).toBeVisible()`
3. Second view -- should show error:
   - `await page.goto(secret.url)`
   - `await expect(page.getByRole('heading', { name: /Secret (Not Available|Already Viewed)/ })).toBeVisible()`

**Test 2: "invalid link shows error"**
1. Navigate to a fabricated URL with a valid-looking but nonexistent ID: `await page.goto('/secret/xxxxxxxxxxxxxxxxxxx01#somefakekey123456789012345678901234567890123')`
   - The ID must be exactly 21 characters (nanoid length). Use `'xxxxxxxxxxxxxxxxxxx01'` (21 chars).
   - The key fragment should be a plausible base64url string (43 chars for 256-bit key).
2. Wait for "Secret Not Available" heading: `await expect(page.getByRole('heading', { name: 'Secret Not Available' })).toBeVisible()`

**Test 3: "missing encryption key shows Invalid Link error"**
1. `const secret = await createTestSecret()`
2. Navigate WITHOUT the key fragment: `await page.goto('/secret/' + secret.id)`
3. Wait for "Invalid Link" heading: `await expect(page.getByRole('heading', { name: 'Invalid Link' })).toBeVisible()`

NOTE on expired secrets: The research recommends skipping expired-secret E2E testing because the minimum expiry is 1 hour and server-side integration tests already cover expiration thoroughly. Do NOT attempt to test expired secrets in E2E -- no short expiry option exists without modifying the server.
