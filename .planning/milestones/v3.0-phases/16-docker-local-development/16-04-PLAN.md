2. In `server/src/middleware/security.ts`, change `httpsRedirect` to use `env.FORCE_HTTPS` instead of `process.env.NODE_ENV !== 'production'`:

   - Change `upgradeInsecureRequests: []` (line 56) to `...(env.FORCE_HTTPS ? { upgradeInsecureRequests: [] } : {})`
   - This prevents browsers from auto-upgrading HTTP to HTTPS when FORCE_HTTPS is off
   - Use spread syntax to conditionally include the directive in the directives object

FORCE_HTTPS should come from the orchestrator (docker-compose.yml or render.yaml), not be baked into the image. This allows the same image to work in both local Docker (FORCE_HTTPS=false) and production (FORCE_HTTPS=true).

  <action>
In `server/src/app.ts`, add an API 404 handler between the API routes (line 65, `app.use('/api/secrets', ...)`) and the SPA static assets block (line 68, `const clientDistPath = ...`).

Insert this middleware:

```typescript
// API catch-all: return JSON 404 for any unmatched /api/* request.
// MUST come after all API routes and before the SPA catch-all to prevent
// API requests from falling through to the HTML response.
app.use('/api', (_req, res) => {
  res.status(404).json({ error: 'not_found' });
});
```

This ensures:
- `GET /api/health` still hits the health router (mounted earlier at line 62)
- `GET /api/secrets/:id` still hits the secrets router (mounted at line 65)
- `GET /api/anything-else` returns `{ "error": "not_found" }` with status 404
- The SPA catch-all `app.get('{*path}')` only handles non-API browser routes

Do NOT modify the SPA catch-all itself — the API 404 handler catches all /api/* requests before they reach it.
