**3. Create server/src/services/onboarding.service.ts** — The single function that enrolls a new user in the Loops sequence. Key requirements:
- Accepts `{ email, name, marketingConsent, subscriptionTier }` — no userId parameter (ZK invariant: Loops is an external system, pass only email + properties)
- Derives `firstName` from `name` by splitting on space and taking index 0 (fall back to full name if no space)
- Calls `loops.sendEvent()` with single-object v6 API
- The event name `'registered'` must match the trigger configured in the Loops UI loop

```typescript
import { loops } from '../config/loops.js';

export interface OnboardingUser {
  email: string;
  name: string;
  marketingConsent: boolean;
  subscriptionTier: 'free' | 'pro';
}

/**
 * Enroll a newly registered user in the Loops onboarding sequence.
 *
 * Fires the "registered" event which triggers the Loops loop:
 *   - Welcome email: sends immediately (no consent required)
 *   - Day-3 key features email: requires marketingConsent === true
 *   - Day-7 upgrade prompt: requires marketingConsent === true AND subscriptionTier !== 'pro'
 *
 * Uses sendEvent() which creates/updates the contact atomically — no separate createContact call needed.
 *
 * ZERO-KNOWLEDGE: only email and non-identifying properties passed to Loops.
 * No userId, no secretId. See .planning/INVARIANTS.md.
 */
export async function enrollInOnboardingSequence(user: OnboardingUser): Promise<void> {
  const firstName = user.name.split(' ')[0] ?? user.name;

  await loops.sendEvent({
    email: user.email,
    eventName: 'registered',
    contactProperties: {
      firstName,
      marketingConsent: user.marketingConsent,
      subscriptionTier: user.subscriptionTier,
    },
  });
}
```

**2. server/src/services/billing.service.ts** — Extend `activatePro()` to update the Loops contact when a user upgrades to Pro. This prevents the day-7 upgrade email from reaching paying customers.

Add imports at the top:
```typescript
import { loops } from '../config/loops.js';

```

Then extend `activatePro()`. The function currently only does a DB UPDATE. Add: look up user email by stripeCustomerId (one SELECT), then call loops.updateContact(). Both operations use fire-and-forget `.catch()` on the Loops call — billing must never fail because Loops is down.

```typescript
export async function activatePro(stripeCustomerId: string): Promise<void> {
  // Update subscription tier in DB (existing behavior)
  await db
    .update(users)
    .set({ subscriptionTier: 'pro' })
    .where(eq(users.stripeCustomerId, stripeCustomerId));

  // Sync Pro status to Loops contact so day-7 audience filter can suppress upgrade email.
  // Look up email by stripeCustomerId — same pattern as getOrCreateStripeCustomer.
  // ZK invariant: stripeCustomerId is the lookup key; userId is not in scope here.
  const [proUser] = await db
    .select({ email: users.email })
    .from(users)
    .where(eq(users.stripeCustomerId, stripeCustomerId));

  if (proUser) {
    void loops
      .updateContact({
        email: proUser.email,
        properties: { subscriptionTier: 'pro' },
      })
      .catch((err: unknown) => {
        // Non-critical: log and continue — billing is unaffected if Loops is down
        // ZK invariant: log only err.message — no userId in same log line
        logger.error(
          { err: err instanceof Error ? err.message : String(err) },
          'Loops contact update failed on Pro upgrade',
        );
      });
  }
}
```
