  <what-built>
    Complete GDPR double opt-in email capture flow:
    - Homepage email form (home.ts) wired to POST /api/subscribers
    - In-flight state: "Joining..." button + disabled during request
    - Success: form replaced with "Check your inbox — we sent a confirmation link to [email]..."
    - GET /api/subscribers/confirm?token= transitions subscriber to confirmed, adds to Resend Audience
    - GET /api/subscribers/unsubscribe?token= marks unsubscribed (idempotent)
    - SPA pages: /confirm and /unsubscribe (minimal centered card, no nav/footer)
    - NOINDEX on both /confirm and /unsubscribe (server X-Robots-Tag + client meta robots)
    - marketing_subscribers table: ip_hash (not plain IP), consent_text, consent_at stored
  </what-built>

  <how-to-verify>
    **Setup:** Ensure both dev servers are running (npm run dev:server, npm run dev:client).
    You need real RESEND_AUDIENCE_ID and IP_HASH_SALT values in .env to test real email delivery.
    Alternatively, verify DB rows directly and check the Resend dashboard for audience additions.

    **ECAP-01: Form submission returns success**
    1. Navigate to http://torchsecret.localhost:1355/ (or http://localhost:5173/)
    2. Scroll to the "Stay in the loop" email capture section
    3. Enter a valid email address you can access
    4. Check the consent checkbox
    5. Click the submit button — observe "Joining..." label + disabled state during request
    6. Expected: Form is replaced with "Check your inbox — we sent a confirmation link to [email]. Click it to join the list. Check spam if you don't see it."
    7. Expected: No toast, no page reload — just inline success message

    **ECAP-02: Consent checkbox enforcement**
    1. Enter a valid email address but do NOT check the consent checkbox
    2. Click submit
    3. Expected: Inline error appears "Please check the consent box to continue." — form not submitted

    **ECAP-03: Confirmation email received**
    1. Check inbox for the email submitted in ECAP-01 test
    2. Expected: Email with subject containing "Confirm" from Resend (RESEND_FROM_EMAIL)
    3. Click the "Confirm my email" CTA button in the email
    4. Expected: Browser opens /confirm?token=... showing "You're on the list!" with "Try Torch Secret" button

    **ECAP-04: Unsubscribe flow**
    1. After confirming, check the marketing_subscribers table:
       `SELECT email, status, unsubscribe_token FROM marketing_subscribers WHERE email = 'your@email.com';`
    2. Copy the unsubscribe_token value
    3. Navigate to http://localhost:5173/unsubscribe?token={unsubscribe_token}
    4. Expected: Immediate unsubscribed confirmation, no "Are you sure?" step

    **Expired token edge case**
    1. Navigate to http://localhost:5173/confirm?token=definitely-not-a-real-token
    2. Expected: "Confirmation link expired" page with "Back to homepage" link

    **NOINDEX check**
    1. `curl -sI http://localhost:3000/confirm | grep -i x-robots`
       Expected: X-Robots-Tag: noindex, nofollow
    2. `curl -sI http://localhost:3000/unsubscribe | grep -i x-robots`
       Expected: X-Robots-Tag: noindex, nofollow

    **ECAP-05: DB record check**
    1. `SELECT email, status, consent_text, consent_at, ip_hash FROM marketing_subscribers LIMIT 5;`
    2. Expected: ip_hash is a 64-char hex string (not a dotted IP address)
    3. Expected: consent_text contains "I agree to receive product updates"
    4. Expected: consent_at is a valid timestamp
  </how-to-verify>
