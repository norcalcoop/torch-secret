## Decisions Made
- `auth.api.deleteUser` requires `body: {}` even though all body fields are optional — omitting body causes 500 (Better Auth bug/quirk — Zod validation reads `ctx.body.password` and throws TypeError if body is undefined)
- `beforeDelete` is async and properly awaited by Better Auth before deletion proceeds — differs from `databaseHooks.user.create.after` which must be non-async
- `loops.deleteContact({ email })` only — no userId field (Loops userId is external system ID, not Better Auth userId). ZK safe.
- Explicit `secrets.userId = null` in beforeDelete is defense-in-depth; the FK `onDelete: 'set null'` already handles this, but the explicit null-out covers a race window between beforeDelete and the actual DB delete
