## Decisions Made
- `deleteAccount()` returns `boolean` so the `confirmBtn` click handler can restore the UI (spinner removal + button re-enable) on failure without calling `navigate('/')` prematurely
- Focus goes to `confirmInput` (not `cancelBtn`) on modal mount — user actively chose to open the deletion flow, focusing the input is the correct progressive UX
- `document.removeEventListener('keydown', handleKeydown)` used (not `once: true`) because the Escape handler must survive across all keystrokes until the modal is explicitly closed
