## Notable Implementation Details

**Better Auth body:{} workaround:** `auth.api.deleteUser` requires `body: {}` even though all body fields are optional. Without it, Better Auth reads `ctx.body.password` and throws TypeError. This is documented in SUMMARY-03 key-decisions and in the INVARIANTS.md.

**ESLint workers override:** `workers/**/*.ts` was added to eslint.config.ts with `disableTypeChecked` because Cloudflare Worker globals (`ScheduledController`, `ExecutionContext`) are not in the root tsconfig. This is correct and matches the `*.config.ts` pattern.

**socket.yml issueRules:** The final socket.yml has `issueRules` for `obfuscatedFile` suppressions on ioredis, posthog-js, lucide, and entities — added during Plan 06 after reviewing the initial Socket.dev scan. This deviates from the Plan 02 spec (which said "no issueRules preemptively") but is correct: the rules were added reactively in response to confirmed false positives.

**layout.ts bug fix (Plan 06):** Double-render bug in login navigation was fixed — `updateAuthLink()` previously assigned both `authLink.addEventListener` and `authLink.onclick`, causing two concurrent `navigate('/login')` calls. The `onclick` assignments were removed; only `addEventListener` remains (layout.ts line 110). This was a bonus fix during Plan 06 verification, not in the original scope.
