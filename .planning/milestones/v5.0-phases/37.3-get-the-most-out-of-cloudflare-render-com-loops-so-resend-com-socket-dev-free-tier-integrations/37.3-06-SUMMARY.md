## Bug Fix (discovered during verification)
- **Issue**: Navigating to `/login` rendered the Sign In form twice (stacked)
- **Root cause**: `layout.ts` `authLink` had both a permanent `addEventListener('click')` AND dynamically-assigned `authLink.onclick` handlers — both fired on click, causing two `navigate('/login')` calls → two concurrent `handleRoute()` executions → race condition where both async renders appended to the same container
- **Fix**: Removed the three redundant `onclick` assignments from `updateAuthLink()` — the `addEventListener` already reads `authLink.href` at click time
- **Commit**: `f1bb837`
