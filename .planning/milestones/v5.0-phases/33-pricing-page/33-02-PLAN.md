Insert this block after the closing `</script>` of the WebApplication JSON-LD and before `</head>`:

```html

            }
          }
        ]
      }
    </script>
```

**Why static in index.html (not dynamic JS injection):** The `PRICE-05` success criterion requires the FAQPage JSON-LD to be verifiable via `curl http://localhost:3000/pricing`. Curl does not execute JavaScript. Dynamic injection via `renderPricingPage()` would NOT satisfy the curl check. The only viable approach for a SPA is static placement in the HTML shell (same as the existing `WebApplication` JSON-LD block). The Express SPA catch-all's `replaceAll('__CSP_NONCE__', ...)` automatically replaces ALL occurrences in the template — no change needed in `server/src/app.ts`.

**CSP nonce:** The new `<script>` tag MUST have `nonce="__CSP_NONCE__"` — the same as the existing JSON-LD block. Helmet's `scriptSrc` directive applies to all `<script>` elements regardless of type attribute. Without the nonce, the browser console will show a CSP violation.

Do NOT add `/pricing` to `NOINDEX_PREFIXES` in `server/src/app.ts` — it is NOT in that array and must remain absent so the pricing page is indexable.
