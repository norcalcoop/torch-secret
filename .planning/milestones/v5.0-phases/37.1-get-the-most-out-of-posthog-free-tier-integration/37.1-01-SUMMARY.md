## Decisions Made

- `protectionType` is a required (not optional) third parameter on `captureSecretCreated` — forces all callers to provide protection context explicitly rather than defaulting silently
- `identifyUser` optional params keep backwards compatibility — existing callers (login.ts, register.ts) continue working unchanged
- `captureSubscriptionActivated` calls `setPersonProperties({ tier: 'pro' })` inline in the same function — single call pattern, no re-identify
- No-op tests use `vi.resetModules()` + dynamic `import()` to get a fresh module with `_initialized = false`; this is cleaner than mocking the env key as undefined (which would still call posthog.init and could leave partial state)
