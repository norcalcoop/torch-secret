**Existing OAuth implementation (no changes needed):**

`client/src/pages/login.ts` — OAuth button click handler:
```typescript
button.addEventListener('click', () => {
  sessionStorage.setItem('oauth_login_provider', provider);
  void authClient.signIn.social({
    provider,
    callbackURL: '/dashboard',
    errorCallbackURL: '/login?error=oauth',
  });
});
```

`client/src/pages/dashboard.ts` — post-OAuth analytics:
```typescript
const oauthLoginProvider = sessionStorage.getItem('oauth_login_provider');
if (oauthLoginProvider === 'google' || oauthLoginProvider === 'github') {
  sessionStorage.removeItem('oauth_login_provider');
  captureUserLoggedIn(oauthLoginProvider);
}
```

**Test location:** `server/src/tests/auth.test.ts` — AUTH-06 and AUTH-07 at lines 280-311.
These tests skip gracefully when credentials are absent (`if (!process.env.GOOGLE_CLIENT_ID) { return; }`).
Once credentials are injected via Infisical, the guard condition is false and the tests run.

**Common failure modes to watch for:**
- `redirect_uri_mismatch` error on Google consent page → check Authorized redirect URIs in Google Cloud Console (exact match, no trailing slash)
- GitHub error "redirect_uri" mismatch → check the "Torch Secret Dev" app callback URL in GitHub Settings
- Error redirect to `/login?error=oauth` → check server logs for the underlying error (likely missing `user:email` scope or account linking failure)
- Session cookie not set after OAuth callback → `sameSite: 'lax'` is already set in auth.ts (correct); this should not occur

**Dev server start command:** `npm run dev:server` (in one terminal) + `npm run dev:client` (in another terminal)
The Vite proxy forwards `/api/auth/*` to Express — OAuth callbacks land on Express correctly via the proxy.

**Portless dev URL:** `http://torchsecret.localhost:1355`
