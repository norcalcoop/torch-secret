  <name>Task 1: Create Infisical project, import secrets, and configure machine identity</name>
  <files>none (external platform setup — no repo files modified)</files>
  <action>
Complete these steps in order. Collect the values marked SAVE — they are needed for Plan 02.

STEP 1 — Create Infisical project
1. Go to https://app.infisical.com and sign in (create free account if needed)
2. Create a new project named "torch-secret"
3. Note the project slug from the URL or Settings tab [SAVE: project-slug]
4. Note the workspace ID (project settings page) [SAVE: workspaceId]
5. Verify three environments exist: dev, staging, prod (Infisical creates them by default)
6. Confirm environment slugs are exactly dev, staging, prod (lowercase)

STEP 2 — Import secrets into dev environment
1. Ensure the local .env file has all real values populated
2. Run: infisical login
3. Run: infisical secrets set --env=dev --path=/ --file=.env
4. Verify in Infisical UI: dev environment shows ~20 secrets

STEP 3 — Populate staging environment
1. Copy all secrets from dev to staging (use Infisical UI "Copy from..." or add manually)
2. Override DATABASE_URL in staging to: postgresql://secureshare:secureshare@db:5432/secureshare
   (Uses Docker-internal service hostname "db" instead of "localhost")

STEP 4 — Populate prod environment
1. Add prod-specific secret values (live Stripe keys, live Resend key, live Loops key, etc.)
2. Ensure VITE_POSTHOG_KEY and VITE_POSTHOG_HOST are in prod (needed at Render build time — Pitfall 9)
3. Add BETTER_AUTH_TRUSTED_ORIGINS to all three environments (missing from .env.example — pre-existing gap):
   - dev: http://torchsecret.localhost:1355
   - staging: http://localhost:5173
   - prod: https://torchsecret.com

STEP 5 — Create machine identity for CI (Universal Auth)
1. Infisical: Organization Settings > Access Control > Machine Identities > Add Identity
2. Name: github-actions-ci, Auth method: Universal Auth
3. After creating, open the identity and add a Client Secret (copy immediately — shown once)
4. [SAVE: INFISICAL_CLIENT_ID and INFISICAL_CLIENT_SECRET]
5. Grant this identity access to the torch-secret project with Developer role

STEP 6 — Add credentials to GitHub Secrets
1. GitHub repo > Settings > Secrets and variables > Actions > New repository secret
2. Add: INFISICAL_CLIENT_ID = (Client ID from Step 5)
3. Add: INFISICAL_CLIENT_SECRET = (Client Secret from Step 5)

STEP 7 — Configure Render App Connection + Secret Sync for prod
1. Render Dashboard > Account Settings > API Keys > Create API Key named "infisical-sync" [SAVE]
2. Infisical > Project > Integrations > App Connections > Add > Render (paste Render API key)
3. Infisical > Project > Integrations > Secret Syncs > Add Sync > Render:
   - Source: prod environment, path /
   - Destination: select the torch-secret Render service
   - Enable Auto-Sync: ON, Save
4. Secret Syncs dashboard should show status "Synced"

STEP 8 — Install Infisical CLI locally (if not already installed)
   macOS: brew install infisical/get-cli/infisical
   Authenticate: infisical login (opens browser)

STEP 9 — Smoke-test dev injection (using workspaceId from Step 1 before .infisical.json exists)
   infisical run --projectId=WORKSPACE_ID --env=dev -- env | grep BETTER_AUTH_SECRET
   Expected: BETTER_AUTH_SECRET=your-actual-secret (not a placeholder)
