## Decisions Made

- `RESEND_FROM_EMAIL` kept in job-level env — it is configuration (a display string), not a secret, and is CI-specific. Infisical holds the production value; CI uses its own branding string.
- `NODE_ENV` kept in job-level env — test suite assertions check `NODE_ENV === 'test'`; Infisical dev environment sets `NODE_ENV=development`, which would break test-mode guards.
- Infisical secrets-action placed after `actions/checkout@v4` and before `actions/setup-node@v4` — secrets available to node setup and all subsequent steps including migrations and test runner.
