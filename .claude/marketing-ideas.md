# Marketing Ideas for Torch Secret

_Last updated: 2026-02-20_

## Top 5 Ideas (Pre-Launch Priority)

### 1. Show HN — Highest-Leverage Launch Channel

**Why it fits:** Hacker News is the densest concentration of the exact persona — developers who understand "encryption key in the URL fragment" and will appreciate it rather than glaze over. The zero-knowledge architecture is the kind of technical story HN rewards. onetimesecret.com got traction there years ago; the "we did it better, here's why" angle is credible.

**How to start:**

1. Write a Show HN post leading with the technical differentiator: _"Show HN: Torch Secret — zero-knowledge one-time secrets (key never reaches the server)"_
2. In the body, explain the RFC 3986 URL fragment mechanic in 2–3 sentences — proof point, not a marketing claim
3. Link the open source repo alongside the live tool — technical users want to verify

**Expected outcome:** 50–500 upvotes if framing lands; organic SEO backlinks, GitHub stars, early signups from high-quality users

**Resources:** ~2 hours to write the post well. Time for 8–10am ET on a weekday.

---

### 2. Competitor Comparison Pages

**Why it fits:** "onetimesecret alternative", "privnote alternative", "secure password sharing tool" are high-intent searches with real volume. The technical advantage is concrete and verifiable — competitors can't match it without rebuilding their architecture.

**How to start:**

1. Create `/alternatives/onetimesecret`, `/alternatives/privnote`, `/alternatives/password-pusher`
2. Lead with the single differentiator: client-side encryption vs. server-side. Show the code path. Show the RFC reference.
3. Feature comparison table: zero-knowledge | open source | no recipient account required | PADME padding

**Expected outcome:** Long-tail SEO traffic from users actively evaluating options — high-conversion visits

**Resources:** 3 pages × ~3 hours each. Evergreen with minimal maintenance.

---

### 3. Powered-By Marketing — Viral Loop in the Product Itself

**Why it fits:** Every secret reveal page is a touchpoint with someone who didn't choose Torch Secret — a recipient who clicked a link. One tasteful line turns every delivery into a marketing event.

**How to start:**

1. Add "Secured with Torch Secret — torchsecret.com" to the reveal page, below the revealed secret
2. Optionally: a one-liner — _"The person who sent this used end-to-end encryption. The server never saw this secret."_
3. Track clicks from that link to understand recipient conversion

**Expected outcome:** Every secret sent is a distribution event. If senders average 2–3 recipients, the user base multiplies passively.

**Resources:** ~1 hour of dev work.

---

### 4. Reddit Marketing — r/netsec, r/devops, r/sysadmin

**Why it fits:** These communities actively discuss credential sharing problems. Skeptical of marketing, receptive to genuine technical contributions. The open source codebase is the credibility badge.

**How to start:**

1. Contribute first, don't promote: answer questions about secure credential sharing for 2–3 weeks to build karma
2. Post a "I built this because…" thread — honest, technical, no hype. GitHub repo first, live tool second.
3. Create an r/netsec explainer post about the URL fragment mechanic — educational content that demonstrates the differentiator

**Expected outcome:** Developer word-of-mouth, GitHub stars, potential security community endorsement

**Resources:** 30–60 min/week. Zero ad spend.

---

### 5. Problem-First SEO Content

**Why it fits:** The audience searches for the problem before the solution. "How to share API keys securely", "send password without email", "share credentials without Slack" have real search volume and weak existing content.

**How to start:**

1. Write 3 cornerstone posts: _"How to share API keys securely"_, _"Why you shouldn't send passwords over Slack"_, _"Zero-knowledge encryption explained for developers"_
2. Each post answers the problem fully, then presents Torch Secret as the practical conclusion — not a pitch
3. Internally link all three to the competitor comparison pages

**Expected outcome:** Compounding organic traffic from high-intent searches over 3–6 months

**Resources:** 3 posts × 4–6 hours each. One-time investment with long-term compounding returns.

---

## Recommended Launch Sequence

| Week        | Action                                                 |
| ----------- | ------------------------------------------------------ |
| Now         | Add powered-by link to reveal page (1 hour, immediate) |
| 1–2         | Write 3 competitor comparison pages                    |
| 2–3         | Begin Reddit community participation                   |
| 3–4         | Write and refine Show HN post draft                    |
| Launch week | Show HN + Reddit simultaneous post                     |
| Post-launch | Start SEO content based on traffic signals             |

The Show HN + Reddit combo is the highest-impact launch play. Competitor pages and powered-by marketing are set-and-forget distribution multipliers that compound from day one.
